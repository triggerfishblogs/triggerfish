# 术语表

| 术语 | 定义 |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **智能体团队** | 具有不同角色的持久协作智能体会话组。其中一个成员是负责协调工作的领导者。通过 `team_create` 创建，使用生命周期检查进行监控。 |
| **A2UI** | Agent-to-UI 协议，用于将视觉内容从智能体实时推送到 Tide Pool 工作区。 |
| **后台会话** | 为自主任务（定时任务、触发器）生成的会话，以全新的 PUBLIC taint 启动，在隔离的工作区中运行。 |
| **Buoy** | 一款伴侣原生应用（iOS、Android），为智能体提供设备能力，如摄像头、位置、屏幕录制和推送通知。（即将推出。） |
| **分类** | 分配给数据、渠道和接收者的敏感度标签。四个级别：RESTRICTED、CONFIDENTIAL、INTERNAL、PUBLIC。 |
| **定时任务** | 智能体使用标准 cron 表达式语法在指定时间执行的计划重复任务。 |
| **Dive** | 首次运行设置向导（`triggerfish dive`），用于搭建 `triggerfish.yaml`、SPINE.md 和初始配置。 |
| **有效分类** | 用于输出决策的分类级别，计算方式为 `min(channel_classification, recipient_classification)`。 |
| **执行环境** | 智能体的代码工作区，用于在紧密的编写-运行-修复反馈循环中编写、运行和调试代码，与 Plugin 沙箱不同。 |
| **故障转移** | 当前 LLM 提供商因速率限制、服务器错误或超时不可用时，自动回退到备用提供商。 |
| **Gateway** | 长期运行的本地控制面板，通过 WebSocket JSON-RPC 端点管理会话、渠道、工具、事件和智能体进程。 |
| **Hook** | 数据流中的确定性执行点，策略引擎在此评估规则并决定是否允许、阻止或编辑操作。 |
| **溯源** | 出处元数据，跟踪 Triggerfish 处理的每个数据元素的来源、转换和当前位置。 |
| **LlmProvider** | LLM 补全的接口，由每个支持的提供商实现（Anthropic、OpenAI、Google、Local、OpenRouter）。 |
| **MCP** | Model Context Protocol，智能体-工具通信的标准。Triggerfish 的 MCP Gateway 为任何 MCP 服务器添加分类控制。 |
| **禁止降级写入** | 固定的、不可配置的规则：数据只能流向分类级别相同或更高的渠道或接收者。 |
| **NotificationService** | 跨所有已连接渠道投递所有者通知的统一抽象，具有优先级、排队和去重功能。 |
| **Patrol** | 诊断健康检查命令（`triggerfish patrol`），验证 Gateway、LLM 提供商、渠道和策略配置。 |
| **Reef（The）** | 用于发现、安装、发布和管理 Triggerfish 技能的社区技能市场。 |
| **Ripple** | 在支持的渠道之间中继的实时输入指示器和在线状态信号。 |
| **会话** | 具有独立 taint 跟踪的对话状态基本单元。每个会话有唯一 ID、用户、渠道、taint 级别和历史。 |
| **技能** | 包含 `SKILL.md` 文件和可选支持文件的文件夹，无需编写插件即可赋予智能体新能力。 |
| **SPINE.md** | 作为系统提示基础加载的智能体身份和使命文件。定义个性、规则和边界。Triggerfish 的 CLAUDE.md 等价物。 |
| **StorageProvider** | 所有有状态数据流转的统一持久化抽象（键值接口）。实现包括 Memory、SQLite 和企业后端。 |
| **Taint** | 根据会话访问的数据附加到会话的分类级别。Taint 在会话内只能升级，永不降低。 |
| **Tide Pool** | 智能体驱动的可视化工作区，Triggerfish 使用 A2UI 协议渲染交互式内容（仪表板、图表、表单）。 |
| **TRIGGER.md** | 智能体的主动行为定义文件，指定在周期性触发器唤醒期间要检查、监控和执行的内容。 |
| **Webhook** | 接受来自外部服务（GitHub、Sentry 等）事件并触发智能体操作的入站 HTTP 端点。 |
| **团队领导** | 智能体团队中的指定协调者。接收团队目标，分解工作，分配任务给成员，并决定团队何时完成。 |
| **工作区** | 智能体编写和执行自己代码的按智能体文件系统目录，与其他智能体隔离。 |
| **降级写入** | 数据从较高分类级别流向较低分类级别的禁止行为（例如，CONFIDENTIAL 数据发送到 PUBLIC 渠道）。 |
