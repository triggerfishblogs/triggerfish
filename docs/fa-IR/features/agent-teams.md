# تیم‌های عامل

يمكن لعامل‌ها Triggerfish إنشاء فرق دائمة از عامل‌ها متعاونين کار می‌کندون بااً روی مهام
باقدة. هر عضو فريق يحصل روی جلسته ودوره وزمینه محادثته وابزارهاه. يُعين عضو واحد
كـ **قائد** وينسق العمل.

الفرق ازاسبة للمهام المفتوحة التي تستدرد از أدوار متخصصة تعمل بالتوازي: بحث +
تحليل + كتابة، یا بنية + تندرذ + ممراجعه کنیدة.

::: info التوفر
تتدرخواست تیم‌های عامل خطة **Power** ($149/شهر) عند استخدام Triggerfish Gateway.
المستخدمون مفتوحو المصدر الذين يشغلون مفاتيح API خاصة بهم لديهم وصول كامل لفرق
عامل‌ها — هر عضو فريق يستهلك استدخیرخیرً از ارائه‌دهندهك المُكوّن.
:::

## ابزارها

### `team_create`

إنشاء فريق دائم از عامل‌ها يتعاونون روی مهمة.

| الباامل                  | النوع  | الزامی | الوصف                                                           |
| ------------------------ | ------ | ----- | --------------------------------------------------------------- |
| `name`                   | string | بله   | اسم الفريق                                                      |
| `task`                   | string | بله   | هدف الفريق                                                      |
| `members`                | array  | بله   | تعريفات أعضاء الفريق                                             |
| `idle_timeout_seconds`   | number | خیر    | مهلة خمول لهر عضو. پیش‌فرض: 300 (5 دقائق)                       |
| `max_lifetime_seconds`   | number | خیر    | أقصى عمر للفريق. پیش‌فرض: 3600 (ساعة)                           |
| `classification_ceiling` | string | خیر    | سقف طبقه‌بندی روی مستوى الفريق                                      |

### `team_status`

فحص الحالة الحالية لفريق نشط.

### `team_message`

إرسال پیام لعضو فريق محدد.

### `team_disband`

إيقاف فريق وإنهاء تمام نشست‌ها الأعضاء.

## چگونه تعمل الفرق

### الإنشاء

عندما يستدعي عامل `team_create`، يقوم Triggerfish بـ:

1. التحقق از تعريف الفريق (الأدوار، عدد القادة، سقوف طبقه‌بندی)
2. إنشاء نشست عامل بازولة لهر عضو از طریق مصنع الازسق
3. حقن **prompt قائمة الفريق** در prompt النظام لهر عضو
4. إرسال المهمة الیالية للقائد
5. بدء مراقب دورة الحياة الذي يفحص صحة الفريق هر 30 ثانية

### التعاون

أعضاء الفريق يتواصلون با بعضهم باستخدام `sessions_send`. خیر يحتاج عامل المُنشئ
لنقل الپیام‌ها بین الأعضاء.

## مراقبة دورة الحياة

### مهلة الخمول

هر عضو لديه مهلة خمول (پیش‌فرض: 5 دقائق). عند الخمول يتلقى تنبيهاً، وعند ضعف
المهلة يُنهى ويُخطر القائد.

### مهلة العمر

الفرق لها عمر أقصى (پیش‌فرض: ساعة). عند الوصول للحد يتلقى القائد هشداراً با 60
ثانية لإنتاج مخرجات نهائية.

## طبقه‌بندی و Taint

نشست‌ها أعضاء الفريق تتبع نفس قوانین طبقه‌بندی كتمام نشست‌ها الأخرى. سقوف طبقه‌بندی
للأعضاء نمی‌توان أن تتجاوز سقف الفريق.

::: danger SECURITY سقوف طبقه‌بندی الأعضاء نمی‌توان أن تتجاوز سقف الفريق. إذا كان
سقف الفريق `INTERNAL`، نمی‌توان تكوين هر عضو بسقف `CONFIDENTIAL`. :::

## الفرق مقابل عامل‌ها الفرعيين

| الجانب          | عامل فرعي (`subagent`)                      | فريق (`team_create`)                                   |
| --------------- | ------------------------------------------- | ------------------------------------------------------ |
| **العمر**       | مهمة واحدة، يُعيد النتيجة ويخرج             | دائم حتى الحل یا انتهاء المهلة                         |
| **الأعضاء**     | عامل واحد                                   | عامل‌ها متعددون بأدوار مویژگی                              |
| **التفاعل**     | أطلق وانسَ از الأصل                         | الأعضاء يتواصلون بحرية از طریق `sessions_send`             |
| **حالة استفاده** | تفویض مهمة مركزة                         | تعاون متعدد الأدوار باقد                                |
