# தீர்மானிக்கும் திறனுடன் கடந்து-செல்லும் அமைப்பு ஆர்கெஸ்ட்ரேஷன்

ஒரு வழக்கமான கொள்முதல்-செலுத்துதல் பணிப்பாய்வு ஒரு டஜன் அமைப்புகளை தொடுகிறது. கொள்முதல் கோரிக்கை ஒரு தளத்தில் தொடங்குகிறது, மற்றொன்றில் ஒப்புதல் சங்கிலிக்கு திருப்பி விடப்படுகிறது, மூன்றாவதில் விற்பனையாளர் தேடல் தூண்டுகிறது, நான்காவதில் கொள்முதல் ஆணை உருவாக்குகிறது, ஐந்தாவதில் பெறும் செயல்முறை தொடங்குகிறது, ஆறாவதில் இன்வாய்ஸ்களை பொருத்துகிறது, ஏழாவதில் கொடுப்பனவு திட்டமிடுகிறது, மற்றும் எட்டாவதில் அனைத்தையும் பதிவு செய்கிறது. ஒவ்வொரு அமைப்புக்கும் அதன் சொந்த API, அதன் சொந்த புதுப்பிப்பு அட்டவணை, அதன் சொந்த அங்கீகார மாதிரி மற்றும் அதன் சொந்த தோல்வி முறைகள் உள்ளன.

பாரம்பரிய தன்னியக்கம் இதை கடினமான குழாய்களுடன் கையாளுகிறது. படி ஒன்று API A-ஐ அழைக்கிறது, பதிலை பாகுபடுத்துகிறது, படி இரண்டுக்கு ஒரு புலத்தை அனுப்புகிறது. அது வேலை செய்யும் வரை வேலை செய்கிறது. ஒரு விற்பனையாளர் பதிவு எதிர்பார்த்ததை விட சற்று வித்தியாசமான வடிவத்தில் இருக்கிறது. குழாய் வடிவமைக்கப்படாத நிலை குறியீட்டுடன் ஒப்புதல் திரும்பி வருகிறது. API புதுப்பிப்பில் புதிய தேவையான புலம் தோன்றுகிறது. ஒரு உடைந்த படி முழு சங்கிலியையும் உடைக்கிறது.

ஆழமான பிரச்சினை தொழில்நுட்ப எளிதாக உடையும் தன்மை அல்ல. உண்மையான வணிக செயல்முறைகளுக்கு தீர்ப்பு தேவை. இந்த இன்வாய்ஸ் முரண்பாட்டை அதிகரிக்க வேண்டுமா அல்லது தானாக தீர்க்க வேண்டுமா? இந்த விற்பனையாளரின் தாமதமான வழங்கல் முறை ஒப்பந்த மதிப்பாய்வை நியாயப்படுத்துகிறதா? இந்த தீர்மானங்கள் தற்போது மக்களின் மனதில் வாழ்கின்றன, அதாவது தன்னியக்கம் மகிழ்ச்சியான பாதையை மட்டுமே கையாள முடியும்.

## Triggerfish இதை எவ்வாறு தீர்க்கிறது

Triggerfish-இன் பணிப்பாய்வு இயந்திரம் YAML-அடிப்படையிலான பணிப்பாய்வு வரையறைகளை இயக்குகிறது, அவை ஒரே குழாயில் நிர்ணயமான தன்னியக்கத்தை AI தர்க்கத்துடன் கலக்கின்றன. பணிப்பாய்வின் ஒவ்வொரு படியும் அனைத்து Triggerfish செயல்பாடுகளையும் நிர்வகிக்கும் அதே பாதுகாப்பு அமலாக்க அடுக்கு வழியாக செல்கிறது.

### நிர்ணயமான பணிக்கு நிர்ணயமான படிகள்

பணிப்பாய்வு படிக்கு தெரிந்த உள்ளீடு மற்றும் தெரிந்த வெளியீடு இருக்கும்போது, அது ஒரு நிலையான HTTP அழைப்பு, ஷெல் கட்டளை அல்லது MCP கருவி அழைப்பாக இயங்குகிறது. LLM ஈடுபாடு இல்லை, தாமதம் இல்லை, அனுமானச் செலவு இல்லை. பணிப்பாய்வு இயந்திரம் REST APIs-க்கு `call: http`, இணைக்கப்பட்ட எந்த MCP சர்வருக்கும் `call: triggerfish:mcp`, மற்றும் கட்டளை வரி கருவிகளுக்கு `run: shell` ஆதரிக்கிறது.

### தீர்மானங்களுக்கான LLM துணை-முகவர்கள்

பணிப்பாய்வு படிக்கு சூழல் தர்க்கம் தேவைப்படும்போது, இயந்திரம் `call: triggerfish:llm` பயன்படுத்தி உண்மையான LLM துணை-முகவர் அமர்வை உருவாக்குகிறது. இது ஒரே ஒரு வழிகாட்டல்/பதில் அல்ல. துணை-முகவருக்கு Triggerfish-இல் பதிவு செய்யப்பட்ட ஒவ்வொரு கருவியிலும் அணுகல் உள்ளது, இதில் வலை தேடல், நினைவகம், உலாவி தன்னியக்கம் மற்றும் இணைக்கப்பட்ட அனைத்து ஒருங்கிணைப்புகளும் அடங்கும்.

துணை-முகவரின் வெளியீடு நேரடியாக பணிப்பாய்வின் அடுத்த படிக்கு செல்கிறது. தர்க்கத்தின் போது வகைப்படுத்தப்பட்ட தரவை அணுகியிருந்தால், அமர்வு taint தானாக அதிகரிக்கிறது மற்றும் மூல பணிப்பாய்வுக்கு திரும்பப் பரவுகிறது.

### உண்மையான சூழலின் அடிப்படையில் நிபந்தனை கிளைகள்

பணிப்பாய்வு DSL நிபந்தனை ரூட்டிங்குக்கான `switch` தொகுதிகள், தொகுதி செயலாக்கத்திற்கான `for` வளைய தொகுதிகள் மற்றும் பணிப்பாய்வு நிலையை புதுப்பிக்க `set` செயல்பாடுகளை ஆதரிக்கிறது. சிக்கலான நிபந்தனைகளை மதிப்பிடக்கூடிய LLM துணை-முகவர் படிகளுடன் இணைந்து, பணிப்பாய்வு வெறும் புல மதிப்புகளுக்கு பதிலாக உண்மையான வணிக சூழலின் அடிப்படையில் கிளைகளாக பிரியலாம்.

### அமைப்புகள் மாறும்போது சுய-குணமாதல்

ஒரு API அதன் பதில் வடிவத்தை மாற்றியதால் நிர்ணயமான படி தோல்வியடையும்போது, பணிப்பாய்வு நிறுத்தாது. இயந்திரம் தோல்வியடைந்த படியை LLM துணை-முகவருக்கு ஒப்படைக்கலாம், அவர் பிழையை படிக்கிறார், பதிலை ஆராய்கிறார் மற்றும் மாற்று அணுகுமுறையை முயற்சிக்கிறார். ஒரு புதிய தேவையான புலம் சேர்த்த API-ஐ துணை-முகவர் பிழை செய்தியை படித்து கோரிக்கையை சரிசெய்து கையாளுகிறார்.

### முழு சங்கிலி முழுவதும் பாதுகாப்பு

Triggerfish பணிப்பாய்வின் ஒவ்வொரு படியும் எந்த நேரடி கருவி அழைப்பின் அதே கொள்கை அமலாக்க hooks வழியாக செல்கிறது. PRE_TOOL_CALL செயல்படுத்தலுக்கு முன் அனுமதிகளை சரிபார்க்கிறது. POST_TOOL_RESPONSE திரும்பிய தரவை வகைப்படுத்துகிறது. PRE_OUTPUT இலக்கு அனுமதிக்கும் வகைப்படுத்தல் நிலையை விட அதிகமான வகைப்படுத்தலில் எதுவும் அமைப்பை விட்டு வெளியேறாது என்பதை உறுதி செய்கிறது.

பணிப்பாய்வு வரையறையே `classification_ceiling` அமைக்கலாம். INTERNAL-ல் வகைப்படுத்தப்பட்ட வாராந்திர சுருக்க பணிப்பாய்வு CONFIDENTIAL தரவை அணுக முடியாது, அதற்கான சான்றுகள் இருந்தாலும். மட்டு குறியீட்டில் செயல்படுத்தப்படுகிறது, LLM ஒரு வழிகாட்டல் அறிவுரையை மதிக்கும் என்று நம்பி அல்ல.

### Cron மற்றும் Webhook தூண்டுதல்கள்

பணிப்பாய்வுகளுக்கு யாரோ அவற்றை கைமுறையாக தொடங்க வேண்டியதில்லை. திட்டமிடல் செய்பவர் திரும்பி வரும் பணிப்பாய்வுகளுக்கு cron-அடிப்படையிலான தூண்டுதல்களும் நிகழ்வு-இயக்கப்படும் செயல்படுத்தலுக்கான webhook தூண்டுதல்களும் ஆதரிக்கிறது.

## நடைமுறையில் இது எப்படி தெரிகிறது

NetSuite, Coupa, DocuSign மற்றும் Slack முழுவதும் கொள்முதல்-செலுத்துதல் நடத்தும் நடுத்தர சந்தை நிறுவனம் முழு சுழற்சியை கையாளும் Triggerfish பணிப்பாய்வை வரையறுக்கிறது. நிர்ணயமான படிகள் கொள்முதல் ஆணைகளை உருவாக்கவும், ஒப்புதல்களை திருப்பி விடவும் மற்றும் இன்வாய்ஸ்களை பொருத்தவும் API அழைப்புகளை கையாளுகின்றன. LLM துணை-முகவர் படிகள் விதிவிலக்குகளை கையாளுகின்றன.

Coupa தனது API-ஐ புதுப்பித்து புல பெயரை மாற்றும்போது, பணிப்பாய்வின் நிர்ணயமான HTTP படி தோல்வியடைகிறது. இயந்திரம் துணை-முகவருக்கு ஒப்படைக்கிறது அவர் பிழையை படிக்கிறார், மாறிய புலத்தை அடையாளம் காட்டுகிறார் மற்றும் சரியான அளவுருவுடன் மறுமுயற்சி செய்கிறார். மனித தலையீடு இல்லாமல் பணிப்பாய்வு முடிகிறது.
