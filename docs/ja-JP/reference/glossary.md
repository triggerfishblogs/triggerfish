# 用語集

| 用語                         | 定義                                                                                                                                                                     |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Agent Team**               | 異なる役割を持つ協力するエージェントセッションの永続的なグループ。1人のメンバーがリードとして作業を調整する。`team_create`で作成され、ライフサイクルチェックで監視される。 |
| **A2UI**                     | エージェントからTide PoolワークスペースにリアルタイムでビジュアルコンテンツをプッシュするためのAgent-to-UIプロトコル。                                                     |
| **Background Session**       | 自律タスク（cron、トリガー）のために生成されるセッション。フレッシュなPUBLIC Taintで始まり、分離されたワークスペースで実行される。                                         |
| **Buoy**                     | カメラ、位置情報、画面録画、プッシュ通知などのデバイス機能をエージェントに提供するコンパニオンネイティブアプリ（iOS、Android）。（近日公開予定）                           |
| **Classification**           | データ、チャンネル、受信者に割り当てられる機密性ラベル。4つのレベル：RESTRICTED、CONFIDENTIAL、INTERNAL、PUBLIC。                                                          |
| **Cron**                     | 標準のcron式構文を使用して指定した時刻にエージェントが実行する定期的なスケジュールタスク。                                                                                 |
| **Dive**                     | `triggerfish.yaml`、SPINE.md、初期設定をスキャフォールドするファーストラン設定ウィザード（`triggerfish dive`）。                                                           |
| **Effective Classification** | 出力の決定に使用される分類レベル。`min(channel_classification, recipient_classification)`として計算される。                                                                |
| **Exec Environment**         | プラグインサンドボックスとは区別される、緊密なwrite-run-fixフィードバックループでコードを書き、実行し、デバッグするためのエージェントのコードワークスペース。               |
| **Failover**                 | レート制限、サーバーエラー、またはタイムアウトにより現在のプロバイダーが利用できない場合に、別のLLMプロバイダーへの自動フォールバック。                                     |
| **Gateway**                  | WebSocket JSON-RPCエンドポイントを通じてセッション、チャンネル、ツール、イベント、エージェントプロセスを管理する長時間稼働のローカルコントロールプレーン。                  |
| **Hook**                     | ポリシーエンジンがルールを評価して、アクションを許可、ブロック、または編集するかを決定するデータフロー内の確定的な強制ポイント。                                             |
| **Lineage**                  | Triggerfishが処理したすべてのデータ要素の出所、変換、現在の場所を追跡するプロベナンスメタデータ。                                                                          |
| **LlmProvider**              | LLM補完のためのインターフェース。サポートされている各プロバイダー（Anthropic、OpenAI、Google、Local、OpenRouter）によって実装される。                                       |
| **MCP**                      | Model Context Protocol。エージェントとツールの通信のための標準。TriggerfishのMCP Gatewayは任意のMCPサーバーに分類コントロールを追加する。                                   |
| **No Write-Down**            | データは等しいかより高い分類レベルのチャンネルまたは受信者にのみ流れることができるという、固定された変更不可のルール。                                                     |
| **NotificationService**      | 優先度、キューイング、重複排除を伴う、接続されたすべてのチャンネルにわたってオーナー通知を配信するための統一された抽象化。                                                 |
| **Patrol**                   | ゲートウェイ、LLMプロバイダー、チャンネル、ポリシー設定を検証する診断ヘルスチェックコマンド（`triggerfish patrol`）。                                                      |
| **Reef（The）**              | Triggerfishスキルを発見し、インストールし、公開し、管理するためのコミュニティスキルマーケットプレイス。                                                                     |
| **Ripple**                   | サポートされているチャンネルをまたいでリレーされるリアルタイムのタイピングインジケーターとオンラインステータスシグナル。                                                   |
| **Session**                  | 独立したTaint追跡を持つ会話状態の基本単位。各セッションは一意のID、ユーザー、チャンネル、Taintレベル、履歴を持つ。                                                         |
| **Skill**                    | プラグインを書かずにエージェントに新しい能力を与える`SKILL.md`ファイルとオプションのサポートファイルを含むフォルダ。                                                       |
| **SPINE.md**                 | システムプロンプトの基盤として読み込まれるエージェントのIDとミッションファイル。個性、ルール、境界を定義する。TriggerfishのCLAUDE.mdに相当する。                            |
| **StorageProvider**          | すべての状態を持つデータが流れる統一された永続化抽象化（キーバリューインターフェース）。実装にはMemory、SQLite、エンタープライズバックエンドが含まれる。                     |
| **Taint**                    | アクセスしたデータに基づいてセッションに付与される分類レベル。TaintはセッションでのみエスカレートCapture。下がることはない。                                                |
| **Tide Pool**                | Triggerfishがエージェントが駆動するビジュアルワークスペースで、A2UIプロトコルを使用してインタラクティブコンテンツ（ダッシュボード、チャート、フォーム）をレンダリングする。  |
| **TRIGGER.md**               | 定期的なトリガー起動中に何を確認し、監視し、アクションするかを指定するエージェントのプロアクティブな動作定義ファイル。                                                     |
| **Webhook**                  | 外部サービス（GitHub、Sentryなど）からのイベントを受け入れ、エージェントアクションをトリガーするインバウンドHTTPエンドポイント。                                            |
| **Team Lead**                | エージェントチームの指定コーディネーター。チームの目標を受け取り、作業を分解し、タスクをメンバーに割り当て、チームが完了したかどうかを決定する。                           |
| **Workspace**                | エージェントが自身のコードを書いて実行するためのエージェントごとのファイルシステムディレクトリ。他のエージェントから分離されている。                                       |
| **Write-Down**               | より高い分類レベルからより低い分類レベルへのデータの流れの禁止（例：CONFIDENTIALデータがPUBLICチャンネルに送信される）。                                                   |
