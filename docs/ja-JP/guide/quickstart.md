# クイックスタート

このガイドでは、Triggerfishshの最初の5分間 — セットアップウィザードの実行から
チャット可能なAIエージェントの完成まで — を順を追って説明します。

## セットアップウィザードの実行

1コマンドインストーラーを使用した場合、ウィザードはインストール中にすでに実行されています。
再実行または最初からやり直す場合は：

```bash
triggerfish dive
```

ウィザードは8つのステップを案内します：

### ステップ1：LLMプロバイダーを選択

```
ステップ 1/8: LLMプロバイダーを選択してください
  > Triggerfish Gateway — APIキー不要
    Anthropic (Claude)
    OpenAI
    Google (Gemini)
    ローカル (Ollama)
    OpenRouter
```

プロバイダーを選択して認証情報を入力します。Triggerfishshは自動failoverで複数の
プロバイダーをサポートしています。**Triggerfish Gateway**が最も簡単なオプションです
— [ProまたはPowerプラン](/ja-JP/pricing)にサブスクリプションすれば、APIキーの設定
なしで管理型LLMと検索インフラに接続されます。

### ステップ2：エージェントの名前を設定

```
ステップ 2/8: エージェントの名前とパーソナリティを設定
  エージェント名: Reef
  ミッション（1文）: 整理された状態を保ち、情報を提供する
  トーン: > プロフェッショナル  カジュアル  簡潔  カスタム
```

これにより`SPINE.md`ファイルが生成されます — エージェントのシステムプロンプトの基盤です。
`~/.triggerfish/SPINE.md`でいつでも編集できます。

### ステップ3：チャンネルに接続

```
ステップ 3/8: 最初のチャンネルに接続
  > CLI（すでに利用可能）
    WebChat
    Telegram（ボットトークンを入力）
    スキップ
```

メッセージングプラットフォームを選択するか、このステップをスキップしてCLIのみで
開始します。後で`triggerfish.yaml`にチャンネルを追加できます。

### ステップ4：オプションのプラグイン

```
ステップ 4/8: オプションのプラグインをインストール
  > Obsidian
    スキップ
```

ノート取り用のObsidianなどのオプション統合に接続します。

### ステップ5：Google Workspaceに接続（オプション）

OAuth2でGmailm、Calendar、Tasks、Drive、SheetsのためにGoogleアカウントI に接続します。
このステップをスキップして後で`triggerfish connect google`で接続することもできます。

### ステップ6：GitHubに接続（オプション）

Personal Access Tokenを貼り付けてGitHubに接続します。リポジトリ、PR、Issue、Actionsの
ツールをエージェントに提供します。このステップをスキップして後で
`triggerfish connect github`で接続することもできます。

### ステップ7：ウェブ検索の設定

エージェントが使用する検索エンジン（Brave SearchまたはセルフホストのsearXNG）を
選択するか、今はスキップします。

### ステップ8：デーモンを起動

```
ステップ 8/8: デーモンとしてインストールしますか？
  > はい（ログイン時に起動、バックグラウンドで実行）
    いいえ（手動起動のみ）
```

マシンがオンになっている間いつでもエージェントを実行するには「はい」を選択します。

## チャットの開始

ウィザードが完了したら、CLIを通じてすぐにチャットを開始できます：

```bash
triggerfish chat
```

または、メッセージングチャンネルを接続した場合は、そこでエージェントにメッセージを送信します。

### 会話例

```
あなた: どんなことを手伝ってもらえますか？

Reef: 幅広いタスクをお手伝いできます。以下のことが設定されています：

  - 質問への回答と会話
  - カレンダーとスケジュールの管理
  - メールとメッセージの要約
  - ドキュメントの作成と編集
  - サンドボックス環境でのコードの実行
  - 情報のウェブ検索
  - Cronジョブによる定期タスクの自動化

何から始めたいですか？

あなた: 毎日朝7時にモーニングブリーフィングを設定してください

Reef: モーニングブリーフィングのcronジョブを作成しました：

  スケジュール: 毎日午前7:00
  タスク: カレンダーイベント、未読メール、天気予報を含むブリーフィングの準備
  配信: CLI（TelegramやSlackで受け取るには設定にチャンネルを追加してください）

最初のブリーフィングは明朝届きます。/cron list でいつでもcronジョブを管理できます。
```

## セットアップの確認

すべてが正常に動作しているかヘルスチェックを実行します：

```bash
triggerfish patrol
```

```
Triggerfish ヘルスチェック

  Gatewayが実行中（PID 12345、稼働時間 2分）
  LLMプロバイダーに接続済み（Anthropic、Claude Sonnet 4.5）
  チャンネル1つが有効（CLI）
  ポリシーエンジンが読み込まれた（4つのルール）
  3つのskillがインストール済み（3つはバンドル）

総合: 正常
```

## 次のステップ

- **エージェントをカスタマイズ** — `~/.triggerfish/SPINE.md`を編集してエージェントの
  パーソナリティと機能を改善します。[SPINEとトリガー](./spine-and-triggers)をご参照ください。
- **チャンネルを追加** — `triggerfish.yaml`でTelegram、Slack、Discord、WhatsAppを
  接続します。[設定](./configuration)をご参照ください。
- **統合に接続** — Google Workspaceには`triggerfish connect google`、GitHubには
  `triggerfish connect github`。[統合](/ja-JP/integrations/)をご参照ください。
- **プロアクティブ動作を設定** — `~/.triggerfish/TRIGGER.md`を作成してエージェントが
  何を監視するかを指示します。[SPINEとトリガー](./spine-and-triggers)をご参照ください。
- **コマンドを探索** — 利用可能なすべてのCLIとインチャットコマンドを確認します。
  [CLIコマンド](./commands)をご参照ください。
