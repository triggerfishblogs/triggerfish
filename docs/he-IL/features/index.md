# סקירת תכונות

מעבר ל[מודל האבטחה](/he-IL/security/) ו[תמיכת הערוצים](/he-IL/channels/)
שלו, Triggerfish מספק יכולות המרחיבות את סוכן ה-AI שלכם מעבר
לשאלה-ותשובה: משימות מתוזמנות, זיכרון עמיד, גישת רשת, קלט קולי
ומעבר אוטומטי בין מודלים מרובים.

## התנהגות יזומה

### [Cron וטריגרים](./cron-and-triggers)

תזמנו משימות חוזרות עם ביטויי cron סטנדרטיים והגדירו התנהגות ניטור
יזומה דרך `TRIGGER.md`. הסוכן שלכם יכול להעביר תדריכי בוקר, לבדוק
צינורות, לנטר הודעות שלא נקראו ולפעול באופן אוטונומי בלוח זמנים ניתן
להגדרה -- הכל עם אכיפת סיווג וסשנים מבודדים.

### [התראות](./notifications)

שירות מסירת התראות המנתב הודעות בכל הערוצים המחוברים עם רמות
עדיפות, תורים לא-מקוונים וסינון כפילויות. מחליף דפוסי התראות מקומיים
בהפשטה מאוחדת.

## כלי סוכן

### [חיפוש רשת ושליפה](./web-search)

חפשו ברשת ושלפו תוכן דפים. הסוכן משתמש ב-`web_search` למציאת מידע
וב-`web_fetch` לקריאת דפי רשת, עם מניעת SSRF ואכיפת מדיניות על כל
הבקשות היוצאות.

### [זיכרון עמיד](./memory)

זיכרון חוצה-סשנים עם שערי סיווג. הסוכן שומר ומשחזר עובדות, העדפות
והקשר בין שיחות. סיווג הזיכרון נכפה לזיהום הסשן -- ה-LLM אינו יכול
לבחור את הרמה.

### [ניתוח תמונות וראייה](./image-vision)

הדביקו תמונות מהלוח שלכם (Ctrl+V ב-CLI, הדבקה בדפדפן ב-Tidepool)
ונתחו קובצי תמונה בדיסק. הגדירו מודל ראייה נפרד לתיאור אוטומטי של
תמונות כאשר המודל הראשי אינו תומך בראייה.

### [חקירת קוד](./explore)

הבנת קוד מובנית באמצעות תת-סוכנים מקבילים. כלי `explore` ממפה עצי
ספריות, מזהה דפוסי קוד, עוקב אחר ייבואים ומנתח היסטוריית git -- הכל
במקביל.

### [ניהול סשנים](./sessions)

בדקו, תקשרו עם ויצרו סשנים. הסוכן יכול להאציל משימות רקע, לשלוח
הודעות חוצות-סשנים ולפנות בערוצים שונים -- הכל תחת אכיפת
אין-כתיבה-למטה.

### [מצב תכנון ומעקב משימות](./planning)

תכנון מובנה לפני יישום (מצב תכנון) ומעקב משימות עמיד (todos) בין סשנים.
מצב תכנון מגביל את הסוכן לחקירה בקריאה בלבד עד שהמשתמש מאשר את
התוכנית.

### [מערכת קבצים ו-Shell](./filesystem)

קראו, כתבו, חפשו והריצו פקודות. הכלים הבסיסיים לפעולות קבצים, עם
תיחום מרחב עבודה ואכיפת רשימת פקודות אסורות.

### [תת-סוכנים ומשימות LLM](./subagents)

האצילו עבודה לתת-סוכנים אוטונומיים או הריצו prompts LLM מבודדים
לסיכום, סיווג והיסק ממוקד ללא זיהום השיחה הראשית.

### [צוותי סוכנים](./agent-teams)

צרו צוותים עמידים של סוכנים משתפי פעולה עם תפקידים מתמחים. מוביל
מתאם חברים המתקשרים באופן אוטונומי דרך הודעות בין-סשנים. כולל ניטור
מחזור חיים עם timeouts של חוסר פעילות, מגבלות זמן חיים ובדיקות
תקינות. מתאים ביותר למשימות מורכבות הנהנות ממספר נקודות מבט החוזרות
על עבודת השני.

## אינטראקציה עשירה

### [צינור קול](./voice)

תמיכת דיבור מלאה עם ספקי STT ו-TTS ניתנים להגדרה. השתמשו ב-Whisper
לתמלול מקומי, Deepgram או OpenAI ל-STT בענן, ו-ElevenLabs או OpenAI
לטקסט-לדיבור. קלט קולי עובר דרך אותה אכיפת סיווג ומדיניות כמו טקסט.

### [Tide Pool / A2UI](./tidepool)

מרחב עבודה חזותי מונחה סוכן שבו Triggerfish מציג תוכן אינטראקטיבי --
לוחות מחוונים, תרשימים, טפסים ותצוגות מקדימות של קוד. פרוטוקול A2UI
(Agent-to-UI) דוחף עדכונים בזמן אמת מהסוכן ללקוחות מחוברים.

## ריבוי סוכנים ומודלים

### [ניתוב רב-סוכני](./multi-agent)

נתבו ערוצים, חשבונות או אנשי קשר שונים לסוכנים מבודדים נפרדים, כל
אחד עם SPINE.md, מרחב עבודה, מיומנויות ותקרת סיווג משלו. ה-Slack
של העבודה הולך לסוכן אחד; ה-WhatsApp האישי שלכם הולך לאחר.

### [ספקי LLM ומעבר אוטומטי](./model-failover)

התחברו ל-Anthropic, OpenAI, Google, מודלים מקומיים (Ollama) או
OpenRouter. הגדירו שרשראות מעבר כך שהסוכן שלכם עובר אוטומטית לספק
חלופי כאשר אחד אינו זמין. כל סוכן יכול להשתמש במודל שונה.

### [הגבלת קצב](./rate-limiting)

מגביל קצב בחלון נע שמונע פגיעה במגבלות API של ספקי LLM. עוקב אחר
tokens-per-minute ו-requests-per-minute, משהה קריאות כאשר הקיבולת
אוזלת ומשתלב עם שרשרת המעבר.

## תפעול

### [תיעוד מובנה](./logging)

תיעוד מובנה מאוחד עם רמות חומרה, סיבוב קבצים ופלט כפול ל-stderr ולקובץ.
שורות יומן מתויגות לפי רכיב, סיבוב אוטומטי ב-1 MB, וכלי `log_read`
לגישה להיסטוריית יומנים.

::: info כל התכונות משתלבות עם מודל האבטחה הליבתי. משימות cron מכבדות
תקרות סיווג. קלט קולי נושא זיהום. תוכן Tide Pool עובר דרך וו PRE_OUTPUT.
ניתוב רב-סוכני אוכף בידוד סשנים. שום תכונה אינה עוקפת את שכבת
המדיניות. :::
