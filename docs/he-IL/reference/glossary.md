# מילון מונחים

| מונח                          | הגדרה                                                                                                                                                                         |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Agent Team**                | קבוצה עמידה של סשני סוכנים משתפי פעולה עם תפקידים ייחודיים. חבר אחד הוא המוביל המתאם את העבודה. נוצר באמצעות `team_create`, נצפה עם בדיקות מחזור חיים.                          |
| **A2UI**                      | פרוטוקול Agent-to-UI לדחיפת תוכן ויזואלי מהסוכן למרחב העבודה Tide Pool בזמן אמת.                                                                                              |
| **Background Session**        | סשן שנוצר למשימות אוטונומיות (cron, טריגרים) שמתחיל עם זיהום PUBLIC טרי ורץ במרחב עבודה מבודד.                                                                                |
| **Buoy**                      | אפליקציית מלווה (iOS, Android) המספקת יכולות מכשיר כגון מצלמה, מיקום, הקלטת מסך והתראות push לסוכן. (בקרוב.)                                                                   |
| **Classification**            | תווית רגישות המוקצית לנתונים, ערוצים ונמענים. ארבע רמות: RESTRICTED, CONFIDENTIAL, INTERNAL, PUBLIC.                                                                            |
| **Cron**                      | משימה חוזרת מתוזמנת המבוצעת על ידי הסוכן בזמן מוגדר באמצעות תחביר ביטויי cron סטנדרטי.                                                                                       |
| **Dive**                      | אשף ההגדרה הראשונית (`triggerfish dive`) היוצר שלד ל-`triggerfish.yaml`, SPINE.md ותצורה ראשונית.                                                                              |
| **Effective Classification**  | רמת הסיווג המשמשת להחלטות פלט, המחושבת כ-`min(channel_classification, recipient_classification)`.                                                                              |
| **Exec Environment**          | מרחב העבודה של הסוכן לכתיבה, הרצה וניפוי קוד בלולאת כתיבה-הרצה-תיקון צמודה, נפרד מ-Plugin Sandbox.                                                                           |
| **Failover**                  | מעבר אוטומטי לספק LLM חלופי כאשר הספק הנוכחי אינו זמין עקב הגבלת קצב, שגיאות שרת או חריגת זמן.                                                                               |
| **Gateway**                   | מישור הבקרה המקומי ארוך-הריצה המנהל סשנים, ערוצים, כלים, אירועים ותהליכי סוכן דרך נקודת קצה JSON-RPC ב-WebSocket.                                                              |
| **Hook**                      | נקודת אכיפה דטרמיניסטית בזרימת הנתונים שבה מנוע המדיניות מעריך כללים ומחליט האם לאפשר, לחסום או לצנזר פעולה.                                                                  |
| **Lineage**                   | מטא-נתוני מוצא העוקבים אחר מקור, טרנספורמציות ומיקום נוכחי של כל אלמנט נתונים שמעובד על ידי Triggerfish.                                                                       |
| **LlmProvider**               | הממשק להשלמות LLM, ממומש על ידי כל ספק נתמך (Anthropic, OpenAI, Google, Local, OpenRouter).                                                                                    |
| **MCP**                       | Model Context Protocol, תקן לתקשורת סוכן-כלי. ה-MCP Gateway של Triggerfish מוסיף בקרות סיווג לכל שרת MCP.                                                                      |
| **No Write-Down**             | הכלל הקבוע והלא-ניתן-להגדרה שנתונים יכולים לזרום רק לערוצים או נמענים ברמת סיווג שווה או גבוהה יותר.                                                                          |
| **NotificationService**       | ההפשטה המאוחדת למסירת התראות לבעלים בכל הערוצים המחוברים עם עדיפות, תור ומניעת כפילויות.                                                                                      |
| **Patrol**                    | פקודת בדיקת בריאות אבחנתית (`triggerfish patrol`) המאמתת את ה-gateway, ספקי LLM, ערוצים ותצורת מדיניות.                                                                         |
| **Reef (The)**                | שוק המיומנויות הקהילתי לגילוי, התקנה, פרסום וניהול מיומנויות Triggerfish.                                                                                                     |
| **Ripple**                    | מחווני הקלדה בזמן אמת ואותות סטטוס מקוון המועברים בין ערוצים שבהם יש תמיכה.                                                                                                  |
| **Session**                   | יחידת מצב השיחה הבסיסית עם מעקב זיהום עצמאי. לכל סשן יש מזהה ייחודי, משתמש, ערוץ, רמת זיהום והיסטוריה.                                                                        |
| **Skill**                     | תיקייה המכילה קובץ `SKILL.md` וקבצים תומכים אופציונליים הנותנים לסוכן יכולות חדשות ללא כתיבת plugins.                                                                          |
| **SPINE.md**                  | קובץ זהות ומשימת הסוכן הנטען כבסיס ל-system prompt. מגדיר אישיות, כללים וגבולות. המקבילה ב-Triggerfish ל-CLAUDE.md.                                                             |
| **StorageProvider**           | הפשטת האחסון המאוחדת (ממשק מפתח-ערך) שדרכה זורמים כל הנתונים בעלי המצב. מימושים כוללים Memory, SQLite ו-backends ארגוניים.                                                     |
| **Taint**                     | רמת הסיווג המצורפת לסשן בהתבסס על הנתונים שניגש אליהם. זיהום יכול רק להסלים בתוך סשן, לעולם לא לרדת.                                                                          |
| **Tide Pool**                 | מרחב עבודה ויזואלי מונחה-סוכן שבו Triggerfish מרנדר תוכן אינטראקטיבי (לוחות מחוונים, תרשימים, טפסים) באמצעות פרוטוקול A2UI.                                                    |
| **TRIGGER.md**                | קובץ הגדרת ההתנהגות היזומה של הסוכן, המציין מה לבדוק, לנטר ולפעול עליו במהלך התעוררויות טריגר תקופתיות.                                                                        |
| **Webhook**                   | נקודת קצה HTTP נכנסת המקבלת אירועים משירותים חיצוניים (GitHub, Sentry וכו') ומפעילה פעולות סוכן.                                                                                |
| **Team Lead**                 | המתאם המיועד בצוות סוכנים. מקבל את מטרת הצוות, מפרק עבודה, מקצה משימות לחברים ומחליט מתי הצוות סיים.                                                                           |
| **Workspace**                 | ספריית מערכת קבצים לכל סוכן שבה הסוכן כותב ומריץ את הקוד שלו, מבודדת מסוכנים אחרים.                                                                                          |
| **Write-Down**                | הזרימה האסורה של נתונים מרמת סיווג גבוהה לנמוכה יותר (למשל, נתוני CONFIDENTIAL שנשלחים לערוץ PUBLIC).                                                                          |
