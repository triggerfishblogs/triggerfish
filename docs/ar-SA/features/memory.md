# الذاكرة الدائمة

وكلاء Triggerfish لديهم ذاكرة دائمة عبر الجلسات. يمكن للوكيل حفظ الحقائق
والتفضيلات والسياق الذي ينجو عبر المحادثات وإعادات التشغيل. الذاكرة محمية
بالتصنيف -- لا يستطيع الوكيل قراءة ذكريات فوق مستوى taint جلسته الحالية.

## الأدوات

### `memory_save`

حفظ حقيقة أو تفضيل أو سياق في الذاكرة طويلة الأمد.

| المعامل    | النوع  | مطلوب | الوصف                              |
| ---------- | ------ | ----- | ---------------------------------- |
| `content`  | string | نعم   | ماذا يُحفظ                         |
| `category` | string | لا    | فئة اختيارية (preference, fact)    |

### `memory_search`

البحث في الذاكرة المحفوظة باستخدام بحث نص كامل.

| المعامل | النوع  | مطلوب | الوصف            |
| ------- | ------ | ----- | ---------------- |
| `query` | string | نعم   | نص البحث          |

### `memory_list`

سرد جميع إدخالات الذاكرة المحفوظة.

### `memory_delete`

حذف إدخال ذاكرة بالمعرف.

## بوابة التصنيف

- **الكتابة**: تُجبر إدخالات الذاكرة على مستوى taint الجلسة الحالية
- **القراءة**: تُفلتر استعلامات الذاكرة بـ `canFlowTo`

::: warning SECURITY لا يستطيع LLM اختيار تصنيف الذاكرة. يتم فرض التصنيف على
مستوى taint الجلسة في hook `PRE_TOOL_CALL`. :::

## التخزين

الذاكرة تُخزن عبر `StorageProvider` مع بحث نص كامل SQLite FTS5 للبحث الفعال.
