# استكشاف قاعدة الكود

أداة `explore` تمنح الوكيل فهماً سريعاً ومنظماً لقواعد الكود والأدلة. بدلاً من
استدعاء `read_file` و `list_directory` و `search_files` بالتتابع يدوياً، يستدعي
الوكيل `explore` مرة واحدة ويحصل على تقرير منظم ينتجه وكلاء فرعيون متوازون.

## الأداة

### `explore`

استكشاف دليل أو قاعدة كود لفهم البنية والأنماط والاتفاقيات. للقراءة فقط.

| المعامل | النوع  | مطلوب | الوصف                                                     |
| ------- | ------ | ----- | --------------------------------------------------------- |
| `path`  | string | نعم   | الدليل أو الملف للاستكشاف                                 |
| `focus` | string | لا    | ما الذي تبحث عنه (مثل "auth patterns", "test structure")  |
| `depth` | string | لا    | مدى الشمولية: `shallow`, `standard` (افتراضي), أو `deep`  |

## مستويات العمق

| العمق      | الوكلاء المُنشأون | ما يُحلل                                                |
| ---------- | ---------------- | ------------------------------------------------------- |
| `shallow`  | 2                | شجرة الدليل + ملفات التبعيات                             |
| `standard` | 3-4              | الشجرة + التبعيات + أنماط الكود + التركيز (إذا محدد)     |
| `deep`     | 5-6              | كل ما سبق + تتبع رسم الاستيراد + سجل git               |

## كيف يعمل

أداة الاستكشاف تُنشئ وكلاء فرعيين متوازيين، كل منها يركز على جانب مختلف:

1. **وكيل الشجرة** -- يُعين بنية الدليل
2. **وكيل الملفات** -- يقرأ ملفات التبعيات
3. **وكيل الأنماط** -- يأخذ عينات من ملفات المصدر لكشف أنماط الكتابة
4. **وكيل التركيز** -- يبحث عن الملفات والأنماط المتعلقة باستعلام التركيز
5. **وكيل الاستيراد** (عميق فقط) -- يتتبع رسوم الاستيراد
6. **وكيل Git** (عميق فقط) -- يحلل التعليقات الأخيرة

جميع الوكلاء يعملون بالتزامن.
