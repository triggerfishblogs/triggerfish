# الوكلاء الفرعيون ومهام LLM

يمكن لوكلاء Triggerfish تفويض العمل لوكلاء فرعيين وتشغيل prompts LLM معزولة.
هذا يمكّن العمل المتوازي والاستدلال المركز وتفكيك المهام متعدد الوكلاء.

## الأدوات

### `subagent`

تفويض مهمة لوكيل فرعي مستقل. يعمل الوكيل الفرعي في جلسة معزولة ويُعيد النتيجة
عند الاكتمال.

| المعامل | النوع  | مطلوب | الوصف                          |
| ------- | ------ | ----- | ------------------------------ |
| `task`  | string | نعم   | وصف المهمة للتفويض             |
| `tools` | array  | لا    | أدوات محددة لمنحها للوكيل الفرعي |
| `model` | string | لا    | تجاوز النموذج لهذا الوكيل الفرعي |

### `llm_task`

تشغيل prompt LLM معزول للتلخيص أو التصنيف أو الاستدلال المركز بدون تلويث
المحادثة الرئيسية.

| المعامل  | النوع  | مطلوب | الوصف                        |
| -------- | ------ | ----- | ---------------------------- |
| `prompt` | string | نعم   | prompt LLM للتنفيذ           |
| `model`  | string | لا    | تجاوز النموذج                |

## الأمان

- الوكلاء الفرعيون يرثون taint من الجلسة الأم
- جميع إجراءات الوكيل الفرعي تمر عبر hooks السياسات
- سقوف التصنيف تنطبق
