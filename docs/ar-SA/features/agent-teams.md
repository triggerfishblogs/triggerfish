# فرق الوكلاء

يمكن لوكلاء Triggerfish إنشاء فرق دائمة من وكلاء متعاونين يعملون معاً على مهام
معقدة. كل عضو فريق يحصل على جلسته ودوره وسياق محادثته وأدواته. يُعين عضو واحد
كـ **قائد** وينسق العمل.

الفرق مناسبة للمهام المفتوحة التي تستفيد من أدوار متخصصة تعمل بالتوازي: بحث +
تحليل + كتابة، أو بنية + تنفيذ + مراجعة.

::: info التوفر
تتطلب فرق الوكلاء خطة **Power** ($149/شهر) عند استخدام Triggerfish Gateway.
المستخدمون مفتوحو المصدر الذين يشغلون مفاتيح API خاصة بهم لديهم وصول كامل لفرق
الوكلاء — كل عضو فريق يستهلك استدلالاً من مزودك المُكوّن.
:::

## الأدوات

### `team_create`

إنشاء فريق دائم من وكلاء يتعاونون على مهمة.

| المعامل                  | النوع  | مطلوب | الوصف                                                           |
| ------------------------ | ------ | ----- | --------------------------------------------------------------- |
| `name`                   | string | نعم   | اسم الفريق                                                      |
| `task`                   | string | نعم   | هدف الفريق                                                      |
| `members`                | array  | نعم   | تعريفات أعضاء الفريق                                             |
| `idle_timeout_seconds`   | number | لا    | مهلة خمول لكل عضو. افتراضي: 300 (5 دقائق)                       |
| `max_lifetime_seconds`   | number | لا    | أقصى عمر للفريق. افتراضي: 3600 (ساعة)                           |
| `classification_ceiling` | string | لا    | سقف تصنيف على مستوى الفريق                                      |

### `team_status`

فحص الحالة الحالية لفريق نشط.

### `team_message`

إرسال رسالة لعضو فريق محدد.

### `team_disband`

إيقاف فريق وإنهاء جميع جلسات الأعضاء.

## كيف تعمل الفرق

### الإنشاء

عندما يستدعي الوكيل `team_create`، يقوم Triggerfish بـ:

1. التحقق من تعريف الفريق (الأدوار، عدد القادة، سقوف التصنيف)
2. إنشاء جلسة وكيل معزولة لكل عضو عبر مصنع المنسق
3. حقن **prompt قائمة الفريق** في prompt النظام لكل عضو
4. إرسال المهمة الأولية للقائد
5. بدء مراقب دورة الحياة الذي يفحص صحة الفريق كل 30 ثانية

### التعاون

أعضاء الفريق يتواصلون مع بعضهم باستخدام `sessions_send`. لا يحتاج الوكيل المُنشئ
لنقل الرسائل بين الأعضاء.

## مراقبة دورة الحياة

### مهلة الخمول

كل عضو لديه مهلة خمول (افتراضي: 5 دقائق). عند الخمول يتلقى تنبيهاً، وعند ضعف
المهلة يُنهى ويُخطر القائد.

### مهلة العمر

الفرق لها عمر أقصى (افتراضي: ساعة). عند الوصول للحد يتلقى القائد تحذيراً مع 60
ثانية لإنتاج مخرجات نهائية.

## التصنيف و Taint

جلسات أعضاء الفريق تتبع نفس قواعد التصنيف كجميع الجلسات الأخرى. سقوف التصنيف
للأعضاء لا يمكن أن تتجاوز سقف الفريق.

::: danger SECURITY سقوف تصنيف الأعضاء لا يمكن أن تتجاوز سقف الفريق. إذا كان
سقف الفريق `INTERNAL`، لا يمكن تكوين أي عضو بسقف `CONFIDENTIAL`. :::

## الفرق مقابل الوكلاء الفرعيين

| الجانب          | وكيل فرعي (`subagent`)                      | فريق (`team_create`)                                   |
| --------------- | ------------------------------------------- | ------------------------------------------------------ |
| **العمر**       | مهمة واحدة، يُعيد النتيجة ويخرج             | دائم حتى الحل أو انتهاء المهلة                         |
| **الأعضاء**     | وكيل واحد                                   | وكلاء متعددون بأدوار مميزة                              |
| **التفاعل**     | أطلق وانسَ من الأصل                         | الأعضاء يتواصلون بحرية عبر `sessions_send`             |
| **حالة الاستخدام** | تفويض مهمة مركزة                         | تعاون متعدد الأدوار معقد                                |
