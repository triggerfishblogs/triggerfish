# مزودو LLM وتجاوز الفشل

يدعم Triggerfish مزودي LLM متعددين مع تجاوز فشل تلقائي واختيار نموذج لكل وكيل
وتبديل نموذج على مستوى الجلسة. بدون قفل على مزود واحد.

## المزودون المدعومون

| المزود     | نقطة النهاية | المصادقة | ملاحظات                          |
| ---------- | ------------ | -------- | -------------------------------- |
| Anthropic  | API          | مفتاح   | Claude Sonnet, Opus, Haiku       |
| OpenAI     | API          | مفتاح   | GPT-4o, o1, o3                   |
| Google     | API          | مفتاح   | Gemini Pro, Flash                |
| Ollama     | محلي         | لا يوجد  | أي نموذج محلي                    |
| LM Studio  | محلي         | لا يوجد  | أي نموذج محلي                    |
| OpenRouter | API          | مفتاح   | 100+ نموذج عبر مزود واحد        |

## سلسلة تجاوز الفشل

```yaml
models:
  primary:
    provider: anthropic
    model: claude-sonnet-4-5
  failover:
    - openai
    - google
```

عندما يفشل المزود الرئيسي (خطأ شبكة، حد معدل، خطأ 500)، يحاول Triggerfish
تلقائياً المزود التالي في السلسلة.

## تبديل النموذج لكل جلسة

```
/model gpt-4o
```

بدّل النموذج للجلسة الحالية. يعود للرئيسي عند إعادة تعيين الجلسة.

## التحقق من المزود

```bash
triggerfish healthcheck
```

يتحقق من اتصال المزود المُكوّن.
