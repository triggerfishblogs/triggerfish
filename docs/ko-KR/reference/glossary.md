# 용어집

| 용어                         | 정의                                                                                                                                                              |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Agent Team**               | 고유 역할을 가진 영구 협업 에이전트 세션 그룹. 한 구성원이 작업을 조정하는 리드. `team_create`로 생성, 수명 주기 검사로 모니터링.                                   |
| **A2UI**                     | 에이전트에서 Tide Pool 워크스페이스로 시각적 콘텐츠를 실시간으로 푸시하기 위한 Agent-to-UI 프로토콜.                                                                |
| **Background Session**       | 자율 작업(cron, 트리거)을 위해 생성된 세션. 새로운 PUBLIC taint로 시작하며 격리된 워크스페이스에서 실행.                                                            |
| **Buoy**                     | 카메라, 위치, 화면 녹화, 푸시 알림 등의 기기 기능을 에이전트에 제공하는 컴패니언 네이티브 앱 (iOS, Android). (출시 예정.)                                           |
| **Classification**           | 데이터, 채널, 수신자에 할당되는 민감도 라벨. 네 가지 수준: RESTRICTED, CONFIDENTIAL, INTERNAL, PUBLIC.                                                             |
| **Cron**                     | 표준 cron 표현식 구문을 사용하여 지정된 시간에 에이전트가 실행하는 예약된 반복 작업.                                                                                |
| **Dive**                     | `triggerfish.yaml`, SPINE.md, 초기 구성을 스캐폴딩하는 최초 실행 설정 마법사 (`triggerfish dive`).                                                                 |
| **Effective Classification** | 출력 결정에 사용되는 분류 수준. `min(channel_classification, recipient_classification)`으로 계산.                                                                  |
| **Exec Environment**         | 긴밀한 쓰기-실행-수정 피드백 루프에서 코드를 작성, 실행, 디버그하기 위한 에이전트의 코드 워크스페이스. Plugin Sandbox와 구별됨.                                     |
| **Failover**                 | 속도 제한, 서버 오류, 타임아웃으로 인해 현재 제공자를 사용할 수 없을 때 대체 LLM 제공자로의 자동 전환.                                                             |
| **Gateway**                  | WebSocket JSON-RPC 엔드포인트를 통해 세션, 채널, 도구, 이벤트, 에이전트 프로세스를 관리하는 장기 실행 로컬 제어 플레인.                                             |
| **Hook**                     | 정책 엔진이 규칙을 평가하고 동작의 허용, 차단, 수정 여부를 결정하는 데이터 흐름의 결정적 시행 지점.                                                                |
| **Lineage**                  | Triggerfish가 처리하는 모든 데이터 요소의 출처, 변환, 현재 위치를 추적하는 프로비넌스 메타데이터.                                                                   |
| **LlmProvider**              | 각 지원 제공자(Anthropic, OpenAI, Google, Local, OpenRouter)가 구현하는 LLM 완성 인터페이스.                                                                       |
| **MCP**                      | Model Context Protocol, 에이전트-도구 통신을 위한 표준. Triggerfish의 MCP Gateway는 모든 MCP 서버에 분류 제어를 추가.                                               |
| **No Write-Down**            | 데이터가 동일하거나 더 높은 분류 수준의 채널 또는 수신자에게만 흐를 수 있다는 고정된, 구성 불가능한 규칙.                                                           |
| **NotificationService**      | 우선순위, 큐잉, 중복 제거를 통해 연결된 모든 채널에서 소유자 알림을 전달하는 통합 추상화.                                                                          |
| **Patrol**                   | gateway, LLM 제공자, 채널, 정책 구성을 확인하는 진단 헬스 체크 명령 (`triggerfish patrol`).                                                                        |
| **Reef (The)**               | Triggerfish skill을 검색, 설치, 게시, 관리하기 위한 커뮤니티 skill 마켓플레이스.                                                                                   |
| **Ripple**                   | 지원되는 채널에서 릴레이되는 실시간 타이핑 표시기 및 온라인 상태 신호.                                                                                             |
| **Session**                  | 독립적인 taint 추적을 가진 대화 상태의 기본 단위. 각 세션은 고유 ID, 사용자, 채널, taint 수준, 기록을 가짐.                                                        |
| **Skill**                    | 플러그인을 작성하지 않고도 에이전트에 새로운 기능을 부여하는 `SKILL.md` 파일과 선택적 지원 파일이 포함된 폴더.                                                      |
| **SPINE.md**                 | 시스템 프롬프트 기반으로 로드되는 에이전트 아이덴티티 및 미션 파일. 성격, 규칙, 경계를 정의. Triggerfish의 CLAUDE.md에 해당.                                        |
| **StorageProvider**          | 모든 상태 데이터가 흐르는 통합 영속성 추상화 (키-값 인터페이스). Memory, SQLite, 엔터프라이즈 백엔드 구현 포함.                                                    |
| **Taint**                    | 세션이 접근한 데이터를 기반으로 세션에 부착되는 분류 수준. 세션 내에서 taint는 상승만 가능하고 하강 불가.                                                           |
| **Tide Pool**                | A2UI 프로토콜을 사용하여 Triggerfish가 대화형 콘텐츠(대시보드, 차트, 양식)를 렌더링하는 에이전트 구동 시각적 워크스페이스.                                          |
| **TRIGGER.md**               | 주기적 트리거 웨이크업 중 확인, 모니터링, 동작할 내용을 지정하는 에이전트의 능동적 동작 정의 파일.                                                                  |
| **Webhook**                  | 외부 서비스(GitHub, Sentry 등)로부터 이벤트를 수신하고 에이전트 동작을 트리거하는 인바운드 HTTP 엔드포인트.                                                         |
| **Team Lead**                | 에이전트 팀의 지정된 조정자. 팀 목표를 수신하고, 작업을 분해하고, 구성원에게 작업을 할당하고, 팀 완료 시점을 결정.                                                  |
| **Workspace**                | 에이전트가 자체 코드를 작성하고 실행하는, 다른 에이전트와 격리된 에이전트별 파일시스템 디렉터리.                                                                    |
| **Write-Down**               | 더 높은 분류 수준에서 더 낮은 분류 수준으로의 금지된 데이터 흐름 (예: CONFIDENTIAL 데이터가 PUBLIC 채널로 전송).                                                    |
