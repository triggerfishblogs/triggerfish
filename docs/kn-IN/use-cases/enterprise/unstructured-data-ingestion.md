# ಅಸಂರಚಿತ ಮತ್ತು ಅರ್ಧ-ಸಂರಚಿತ ಡೇಟಾ ಸ್ವೀಕೃತಿ

ಇನ್‌ವಾಯ್ಸ್ ಸಂಸ್ಕರಣೆ ಈಗಾಗಲೇ ಪರಿಹರಿಸಲ್ಪಟ್ಟ ಸಮಸ್ಯೆಯಾಗಿರಬೇಕಿತ್ತು. ದಾಖಲೆ ಬರುತ್ತದೆ, ಕ್ಷೇತ್ರಗಳನ್ನು ಹೊರತೆಗೆಯಲಾಗುತ್ತದೆ, ಡೇಟಾ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ದಾಖಲೆಗಳ ವಿರುದ್ಧ ಮೌಲ್ಯೀಕರಿಸಲಾಗುತ್ತದೆ, ಮತ್ತು ಫಲಿತಾಂಶ ಸರಿಯಾದ ವ್ಯವಸ್ಥೆಗೆ ರೂಟ್ ಮಾಡಲಾಗುತ್ತದೆ. ವಾಸ್ತವ ಏನೆಂದರೆ ಇನ್‌ವಾಯ್ಸ್ ಸಂಸ್ಕರಣೆ ಮಾತ್ರ ಎಂಟರ್‌ಪ್ರೈಸ್‌ಗಳಿಗೆ ವಾರ್ಷಿಕ ಶತಕೋಟಿ ಮ್ಯಾನ್ಯುಯಲ್ ಶ್ರಮದ ವೆಚ್ಚ ತಗಲುತ್ತದೆ, ಮತ್ತು ಅದನ್ನು ಸರಿಪಡಿಸಲು ವಿನ್ಯಾಸಿಸಿದ ಸ್ವಯಂಚಾಲನ ಯೋಜನೆಗಳು ನಿರಂತರವಾಗಿ ಮುರಿಯುತ್ತಿರುತ್ತವೆ.

ಕಾರಣ ಸ್ವರೂಪ ವ್ಯತ್ಯಾಸ. ಇನ್‌ವಾಯ್ಸ್‌ಗಳು PDF, ಇಮೇಲ್ ಲಗತ್ತುಗಳು, ಸ್ಕ್ಯಾನ್ ಮಾಡಿದ ಚಿತ್ರಗಳು, ಸ್ಪ್ರೆಡ್‌ಶೀಟ್ ಎಕ್ಸ್‌ಪೋರ್ಟ್‌ಗಳು, ಮತ್ತು ಕೆಲವೊಮ್ಮೆ ಫ್ಯಾಕ್ಸ್‌ಗಳಾಗಿ ಬರುತ್ತವೆ. ಪ್ರತಿ ವೆಂಡರ್ ಬೇರೆ ಲೇಔಟ್ ಬಳಸುತ್ತದೆ. ಲೈನ್ ಐಟಂಗಳು ಕೋಷ್ಟಕಗಳಲ್ಲಿ, ಮುಕ್ತ ಪಠ್ಯದಲ್ಲಿ, ಅಥವಾ ಎರಡರ ಸಂಯೋಜನೆಯಲ್ಲಿ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತವೆ. ತೆರಿಗೆ ಲೆಕ್ಕಾಚಾರಗಳು ನ್ಯಾಯವ್ಯಾಪ್ತಿಯ ಪ್ರಕಾರ ಬೇರೆ ನಿಯಮಗಳನ್ನು ಅನುಸರಿಸುತ್ತವೆ. ಅದೇ ವೆಂಡರ್ ಸಹ ಸೂಚನೆ ಇಲ್ಲದೆ ತಮ್ಮ ಇನ್‌ವಾಯ್ಸ್ ಟೆಂಪ್ಲೇಟ್ ಬದಲಾಯಿಸುತ್ತಾರೆ.

ಸಾಂಪ್ರದಾಯಿಕ RPA ಇದನ್ನು ಟೆಂಪ್ಲೇಟ್ ಹೊಂದಾಣಿಕೆಯಿಂದ ನಿರ್ವಹಿಸುತ್ತದೆ. ಇನ್‌ವಾಯ್ಸ್ ಸಂಖ್ಯೆ ಎಲ್ಲಿ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತದೆ, ಲೈನ್ ಐಟಂಗಳು ಎಲ್ಲಿ ಪ್ರಾರಂಭವಾಗುತ್ತವೆ, ಒಟ್ಟು ಮೊತ್ತ ಎಲ್ಲಿದೆ ಎಂದು ನಿರ್ದೇಶಾಂಕಗಳನ್ನು ವ್ಯಾಖ್ಯಾನಿಸಿ. ಒಂದೇ ವೆಂಡರ್‌ನ ಪ್ರಸ್ತುತ ಟೆಂಪ್ಲೇಟ್‌ಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ನಂತರ ವೆಂಡರ್ ತಮ್ಮ ವ್ಯವಸ್ಥೆ ನವೀಕರಿಸುತ್ತಾರೆ, ಕಾಲಮ್ ಬದಲಾಯಿಸುತ್ತಾರೆ, ಶೀರ್ಷಿಕೆ ಸಾಲು ಸೇರಿಸುತ್ತಾರೆ, ಅಥವಾ PDF ಜನರೇಟರ್ ಬದಲಾಯಿಸುತ್ತಾರೆ, ಮತ್ತು ಬಾಟ್ ಸಂಪೂರ್ಣ ವಿಫಲವಾಗುತ್ತದೆ ಅಥವಾ ತಪ್ಪು ಡೇಟಾ ಹೊರತೆಗೆಯುತ್ತದೆ.

## Triggerfish ಇದನ್ನು ಹೇಗೆ ಪರಿಹರಿಸುತ್ತದೆ

Triggerfish ಸ್ಥಾನಿಕ ಕ್ಷೇತ್ರ ಹೊರತೆಗೆಯುವಿಕೆಯನ್ನು LLM-ಆಧಾರಿತ ದಾಖಲೆ ಅರಿವಿನಿಂದ ಬದಲಾಯಿಸುತ್ತದೆ. AI ದಾಖಲೆಯನ್ನು ಮಾನವ ಮಾಡುವ ರೀತಿಯಲ್ಲಿ ಓದುತ್ತದೆ: ಸಂದರ್ಭ ಅರ್ಥ ಮಾಡಿಕೊಂಡು, ಕ್ಷೇತ್ರಗಳ ನಡುವಿನ ಸಂಬಂಧಗಳನ್ನು ಊಹಿಸಿ, ಮತ್ತು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಲೇಔಟ್ ಬದಲಾವಣೆಗಳಿಗೆ ಹೊಂದಿಕೊಂಡು.

### LLM-ಚಾಲಿತ ದಾಖಲೆ ಪಾರ್ಸಿಂಗ್

ದಾಖಲೆ Triggerfish ವರ್ಕ್‌ಫ್ಲೋ ಪ್ರವೇಶಿಸಿದಾಗ, LLM ಉಪ-ಏಜೆಂಟ್ ಸಂಪೂರ್ಣ ದಾಖಲೆ ಓದಿ ದಾಖಲೆ ಅರ್ಥದ ಆಧಾರದ ಮೇಲೆ ರಚನಾತ್ಮಕ ಡೇಟಾ ಹೊರತೆಗೆಯುತ್ತದೆ. ಇನ್‌ವಾಯ್ಸ್ ಸಂಖ್ಯೆ ಒಂದು ಇನ್‌ವಾಯ್ಸ್ ಸಂಖ್ಯೆ, ಅದು ಮೇಲಿನ-ಬಲ ಮೂಲೆಯಲ್ಲಿ "Invoice #" ಲೇಬಲ್‌ನೊಂದಿಗೆ ಇದ್ದರೂ, ಪಠ್ಯ ಪ್ಯಾರಾಗ್ರಾಫ್‌ನಲ್ಲಿ ಹುದುಗಿಸಿದ್ದರೂ, ಅಥವಾ "Factura No." ಲೇಬಲ್‌ನೊಂದಿಗೆ ಇದ್ದರೂ. LLM ಅರ್ಥ ಮಾಡಿಕೊಳ್ಳುತ್ತದೆ "Net 30" ಪಾವತಿ ನಿಯಮಗಳನ್ನು ಅರ್ಥಿಸುತ್ತದೆ, "Qty" ಮತ್ತು "Quantity" ಮತ್ತು "Units" ಒಂದೇ ಅರ್ಥ ಹೊಂದಿವೆ.

ಇದು "ChatGPT ಗೆ ದಾಖಲೆ ಕಳಿಸಿ ಮತ್ತು ಆಶಿಸಿ" ವಿಧಾನ ಅಲ್ಲ. ವರ್ಕ್‌ಫ್ಲೋ ವ್ಯಾಖ್ಯಾನ LLM ಉತ್ಪಾದಿಸಬೇಕಾದ ರಚನಾತ್ಮಕ ಔಟ್‌ಪುಟ್ ಏನು, ಯಾವ ಮೌಲ್ಯೀಕರಣ ನಿಯಮಗಳು ಅನ್ವಯಿಸುತ್ತವೆ, ಮತ್ತು ಹೊರತೆಗೆಯುವ ವಿಶ್ವಾಸ ಕಡಿಮೆ ಇದ್ದಾಗ ಏನಾಗುತ್ತದೆ ಎಂದು ನಿಖರವಾಗಿ ನಿರ್ದಿಷ್ಟಪಡಿಸುತ್ತದೆ.

### ದಾಖಲೆ ಪಡೆಯಲು ಬ್ರೌಸರ್ ಸ್ವಯಂಚಾಲನ

ಅನೇಕ ದಾಖಲೆ ಸ್ವೀಕೃತಿ ವರ್ಕ್‌ಫ್ಲೋಗಳು ಮೊದಲ ಸ್ಥಾನದಲ್ಲಿ ದಾಖಲೆ ಪಡೆಯುವುದರಿಂದ ಪ್ರಾರಂಭವಾಗುತ್ತವೆ. ವಿಮಾ EOBಗಳು ಪಾವತಿದಾರ ಪೋರ್ಟಲ್‌ಗಳಲ್ಲಿ ಇರುತ್ತವೆ. ವೆಂಡರ್ ಇನ್‌ವಾಯ್ಸ್‌ಗಳು ಸರಬರಾಜುದಾರ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳಲ್ಲಿ ಇರುತ್ತವೆ. ಸರ್ಕಾರಿ ಫಾರಂಗಳು ರಾಜ್ಯ ಏಜೆನ್ಸಿ ವೆಬ್‌ಸೈಟ್‌ಗಳಲ್ಲಿ ಇರುತ್ತವೆ.

Triggerfish ನ ಬ್ರೌಸರ್ ಸ್ವಯಂಚಾಲನ ಪುಟ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್‌ಗಳನ್ನು ಓದಲು LLM ನೊಂದಿಗೆ CDP-ನಿಯಂತ್ರಿತ Chromium ಬಳಸುತ್ತದೆ. ಏಜೆಂಟ್ ಪುಟವನ್ನು ಮಾನವ ಮಾಡುವ ರೀತಿಯಲ್ಲಿ ನೋಡುತ್ತದೆ ಮತ್ತು ಹಾರ್ಡ್‌ಕೋಡ್ ಮಾಡಿದ CSS ಸೆಲೆಕ್ಟರ್‌ಗಳ ಬದಲು ಅದು ನೋಡುವುದರ ಆಧಾರದ ಮೇಲೆ ಕ್ಲಿಕ್, ಟೈಪ್, ಮತ್ತು ಸ್ಕ್ರಾಲ್ ಮಾಡುತ್ತದೆ. ಪಾವತಿದಾರ ಪೋರ್ಟಲ್ ಅದರ ಲಾಗಿನ್ ಪುಟ ಮರು-ವಿನ್ಯಾಸಿಸಿದಾಗ, ಏಜೆಂಟ್ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ ಏಕೆಂದರೆ ಅದು ಇನ್ನೂ ದೃಶ್ಯ ಸಂದರ್ಭದಿಂದ ಬಳಕೆದಾರಹೆಸರು ಕ್ಷೇತ್ರ, ಪಾಸ್‌ವರ್ಡ್ ಕ್ಷೇತ್ರ, ಮತ್ತು ಸಲ್ಲಿಕೆ ಬಟನ್ ಗುರುತಿಸಬಲ್ಲದು.

### ವರ್ಗೀಕರಣ-ಗೇಟ್ ಮಾಡಿದ ಸಂಸ್ಕರಣೆ

ದಾಖಲೆಗಳು ಸಂವೇದನಶೀಲತೆಯ ಬೇರೆ ಮಟ್ಟಗಳನ್ನು ಒಯ್ಯುತ್ತವೆ, ಮತ್ತು ವರ್ಗೀಕರಣ ವ್ಯವಸ್ಥೆ ಇದನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ. ಬೆಲೆ ನಿಗದಿ ಷರತ್ತುಗಳನ್ನು ಒಳಗೊಂಡ ಇನ್‌ವಾಯ್ಸ್ CONFIDENTIAL ಆಗಿರಬಹುದು. PHI ಒಳಗೊಂಡ ದಾಖಲೆ RESTRICTED. LLM ಉಪ-ಏಜೆಂಟ್ ದಾಖಲೆ ಓದಿ ಡೇಟಾ ಹೊರತೆಗೆದಾಗ, POST_TOOL_RESPONSE hook ಹೊರತೆಗೆದ ವಿಷಯ ವರ್ಗೀಕರಿಸುತ್ತದೆ, ಮತ್ತು ಅಧಿವೇಶನ taint ಅದಕ್ಕೆ ಅನುಗುಣವಾಗಿ ಹೆಚ್ಚಾಗುತ್ತದೆ.

CONFIDENTIAL ನಲ್ಲಿ ವರ್ಗೀಕರಿಸಿದ ಹೊರತೆಗೆದ ಇನ್‌ವಾಯ್ಸ್ ಡೇಟಾ PUBLIC ನಲ್ಲಿ ವರ್ಗೀಕರಿಸಿದ Slack ಚಾನೆಲ್‌ಗೆ ಕಳಿಸಲಾಗುವುದಿಲ್ಲ. PHI ಒಳಗೊಂಡ ವಿಮಾ ದಾಖಲೆಗಳನ್ನು ಸಂಸ್ಕರಿಸುವ ವರ್ಕ್‌ಫ್ಲೋ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೊರತೆಗೆದ ಡೇಟಾ ಹೋಗಬಹುದಾದ ಜಾಗವನ್ನು ನಿರ್ಬಂಧಿಸುತ್ತದೆ. write-down ತಡೆ ನಿಯಮ ಪ್ರತಿ ಗಡಿಯಲ್ಲಿ ಇದನ್ನು ಜಾರಿ ಮಾಡುತ್ತದೆ, ಮತ್ತು LLM ಅದನ್ನು ಅತಿಕ್ರಮಿಸಲು ಶೂನ್ಯ ಅಧಿಕಾರ ಹೊಂದಿರುತ್ತದೆ.

### ಸ್ವಯಂ-ಚಿಕಿತ್ಸಾ ಸ್ವರೂಪ ಹೊಂದಾಣಿಕೆ

ವೆಂಡರ್ ತಮ್ಮ ಇನ್‌ವಾಯ್ಸ್ ಟೆಂಪ್ಲೇಟ್ ಬದಲಾಯಿಸಿದಾಗ, ಸಾಂಪ್ರದಾಯಿಕ ಸ್ವಯಂಚಾಲನ ಮುರಿದು ಮುರಿದ ಸ್ಥಿತಿಯಲ್ಲಿ ಉಳಿಯುತ್ತದೆ. Triggerfish ನಲ್ಲಿ, LLM ಉಪ-ಏಜೆಂಟ್ ಮುಂದಿನ ರನ್‌ನಲ್ಲಿ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ. ಅದು ಇನ್ನೂ ಇನ್‌ವಾಯ್ಸ್ ಸಂಖ್ಯೆ, ಲೈನ್ ಐಟಂಗಳು, ಮತ್ತು ಒಟ್ಟು ಮೊತ್ತ ಕಂಡುಕೊಳ್ಳುತ್ತದೆ, ಏಕೆಂದರೆ ಅದು ಸ್ಥಾನಕ್ಕಿಂತ ಅರ್ಥಕ್ಕಾಗಿ ಓದುತ್ತದೆ.

ಕಾಲಾನಂತರದಲ್ಲಿ, ಏಜೆಂಟ್ ಕ್ರಾಸ್-ಅಧಿವೇಶನ ಮೆಮೊರಿ ಬಳಸಿ ಮಾದರಿಗಳನ್ನು ಕಲಿಯಬಲ್ಲದು. ವೆಂಡರ್ A ಯಾವಾಗಲೂ ಇತರ ವೆಂಡರ್‌ಗಳಿಲ್ಲದ ರಿಸ್ಟಾಕಿಂಗ್ ಶುಲ್ಕ ಸೇರಿಸಿದ್ದರೆ, ಏಜೆಂಟ್ ಹಿಂದಿನ ಹೊರತೆಗೆಯುವಿಕೆಯಿಂದ ಅದನ್ನು ನೆನಪಿಸಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಅದನ್ನು ಹುಡುಕಲು ತಿಳಿಯುತ್ತದೆ.

### ಪೈಪ್‌ಲೈನ್ ಆರ್ಕೆಸ್ಟ್ರೇಶನ್

ದಾಖಲೆ ಸ್ವೀಕೃತಿ ವಿರಳವಾಗಿ ಕೇವಲ "ಹೊರತೆಗೆದು ಸಂಗ್ರಹಿಸಿ" ಆಗಿರುತ್ತದೆ. ಸಂಪೂರ್ಣ ಪೈಪ್‌ಲೈನ್ ದಾಖಲೆ ತರುತ್ತದೆ, ರಚನಾತ್ಮಕ ಡೇಟಾ ಹೊರತೆಗೆಯುತ್ತದೆ, ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ದಾಖಲೆಗಳ ವಿರುದ್ಧ ಅದನ್ನು ಮೌಲ್ಯೀಕರಿಸುತ್ತದೆ, ಮಾನವ ವಿಮರ್ಶೆಗಾಗಿ ಅಪವಾದಗಳನ್ನು ರೂಟ್ ಮಾಡುತ್ತದೆ, ಮತ್ತು ಮೌಲ್ಯೀಕರಿಸಿದ ಡೇಟಾ ಗುರಿ ವ್ಯವಸ್ಥೆಗೆ ಲೋಡ್ ಮಾಡುತ್ತದೆ.

ಆರೋಗ್ಯ ರಕ್ಷಣೆ ಪೂರ್ವ ಅನುಮೋದನೆ ಪೈಪ್‌ಲೈನ್ ಹೀಗೆ ಕಾಣಬಹುದು: ಬ್ರೌಸರ್ ಸ್ವಯಂಚಾಲನ ಪ್ರದಾತ ಪೋರ್ಟಲ್‌ನಿಂದ ಫ್ಯಾಕ್ಸ್ ಚಿತ್ರ ತರುತ್ತದೆ, LLM ಉಪ-ಏಜೆಂಟ್ ರೋಗಿ ಗುರುತಿಸುವಿಕೆಗಳು ಮತ್ತು ಪ್ರಕ್ರಿಯೆ ಕೋಡ್‌ಗಳನ್ನು ಹೊರತೆಗೆಯುತ್ತದೆ, HTTP ಕರೆ EHR ಗೆ ವಿರುದ್ಧ ರೋಗಿ ಮೌಲ್ಯೀಕರಿಸುತ್ತದೆ, ಮತ್ತೊಂದು ಉಪ-ಏಜೆಂಟ್ ಅನುಮೋದನೆ ವೈದ್ಯಕೀಯ ಅಗತ್ಯತೆ ಮಾನದಂಡಗಳನ್ನು ಪೂರೈಸುತ್ತದೆಯೇ ಎಂದು ಮೌಲ್ಯಮಾಪನ ಮಾಡುತ್ತದೆ, ಮತ್ತು ಫಲಿತಾಂಶ ಸ್ವಯಂ-ಅನುಮೋದನೆ ಅಥವಾ ವೈದ್ಯಕೀಯ ಸಮೀಕ್ಷಕ ಸರದಿಗೆ ರೂಟ್ ಮಾಡಲಾಗುತ್ತದೆ. ಪ್ರತಿ ಹಂತ ವರ್ಗೀಕರಣ-ಟ್ರ್ಯಾಕ್ ಮಾಡಲಾಗಿದೆ.

## ಅಭ್ಯಾಸದಲ್ಲಿ ಇದು ಹೇಗೆ ಕಾಣುತ್ತದೆ

ಪ್ರಾದೇಶಿಕ ಆರೋಗ್ಯ ವ್ಯವಸ್ಥೆ ನಲವತ್ತು ವಿಭಿನ್ನ ಪ್ರದಾತ ಕಚೇರಿಗಳಿಂದ ಪೂರ್ವ ಅನುಮೋದನೆ ವಿನಂತಿಗಳನ್ನು ಸಂಸ್ಕರಿಸುತ್ತದೆ, ಪ್ರತಿಯೊಂದೂ ತಮ್ಮದೇ ಫಾರಂ ಲೇಔಟ್ ಬಳಸಿ, ಕೆಲವು ಫ್ಯಾಕ್ಸ್, ಕೆಲವು ಇಮೇಲ್, ಕೆಲವು ಪೋರ್ಟಲ್‌ಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿದ. ಸಾಂಪ್ರದಾಯಿಕ ವಿಧಾನಕ್ಕೆ ಎಂಟು ಜನರ ತಂಡ ಅಗತ್ಯ ಇತ್ತು, ಏಕೆಂದರೆ ಯಾವ ಸ್ವಯಂಚಾಲನ ಸಾಧನ ಸ್ವರೂಪ ವ್ಯತ್ಯಾಸವನ್ನು ವಿಶ್ವಸನೀಯವಾಗಿ ನಿರ್ವಹಿಸಲಾಗಲಿಲ್ಲ.

Triggerfish ನೊಂದಿಗೆ, ವರ್ಕ್‌ಫ್ಲೋ ಸಂಪೂರ್ಣ ಪೈಪ್‌ಲೈನ್ ನಿರ್ವಹಿಸುತ್ತದೆ. ಬ್ರೌಸರ್ ಸ್ವಯಂಚಾಲನ ಅಥವಾ ಇಮೇಲ್ ಪಾರ್ಸಿಂಗ್ ದಾಖಲೆಗಳನ್ನು ಹಿಂಪಡೆಯುತ್ತದೆ. LLM ಉಪ-ಏಜೆಂಟ್‌ಗಳು ಸ್ವರೂಪ ಲೆಕ್ಕಿಸದೆ ರಚನಾತ್ಮಕ ಡೇಟಾ ಹೊರತೆಗೆಯುತ್ತವೆ. RESTRICTED ವರ್ಗೀಕರಣ ಮಿತಿ PHI ಯಾವತ್ತೂ ಪೈಪ್‌ಲೈನ್ ಗಡಿ ಬಿಡಲಾಗುವುದಿಲ್ಲ ಎಂದು ಖಚಿತಪಡಿಸುತ್ತದೆ.

ಎಂಟು-ಜನರ ತಂಡ ವ್ಯವಸ್ಥೆ ಫ್ಲ್ಯಾಗ್ ಮಾಡುವ ಅಪವಾದಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಎರಡು ಜನರಾಗುತ್ತಾರೆ, ಜೊತೆಗೆ ಸ್ವಯಂಚಾಲಿತ ಹೊರತೆಗೆಯುವಿಕೆಯ ನಿಯತಕಾಲಿಕ ಗುಣಮಟ್ಟ ಲೆಕ್ಕಪರಿಶೋಧನೆ. ನಿರ್ವಹಣೆ ವೆಚ್ಚ ಬಹುತೇಕ ಶೂನ್ಯಕ್ಕೆ ಇಳಿಯುತ್ತದೆ.
