# ತೃತೀಯ-ಪಕ್ಷ ಪೋರ್ಟಲ್‌ಗಳ ವಿರುದ್ಧ UI-ಅವಲಂಬಿತ ಸ್ವಯಂಚಾಲನ

ಪ್ರತಿ ಎಂಟರ್‌ಪ್ರೈಸ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿರಬೇಕಾದ ಆದರೆ ಅಲ್ಲದ ಕೆಲಸ ಮಾಡಲು ಉದ್ಯೋಗಿಗಳು ಪ್ರತಿದಿನ ಹಸ್ತಚಾಲಿತವಾಗಿ ಲಾಗ್ ಇನ್ ಮಾಡುವ ಪೋರ್ಟಲ್‌ಗಳ ಪಟ್ಟಿ ಹೊಂದಿರುತ್ತದೆ. ಆರ್ಡರ್ ಸ್ಥಿತಿ ಪರಿಶೀಲಿಸಲು ವೆಂಡರ್ ಪೋರ್ಟಲ್‌ಗಳು. ನಿಯಂತ್ರಕ ಸಲ್ಲಿಕೆಗಳು ದಾಖಲಿಸಲು ಸರ್ಕಾರಿ ತಾಣಗಳು. ಅರ್ಹತೆ ಪರಿಶೀಲಿಸಲು ಮತ್ತು ಕ್ಲೈಮ್ ಸ್ಥಿತಿ ಪರಿಶೀಲಿಸಲು ವಿಮಾ ಪಾವತಿದಾರ ಪೋರ್ಟಲ್‌ಗಳು. ರುಜುವಾತು ಪರಿಶೀಲನೆಗೆ ರಾಜ್ಯ ಪರವಾನಗಿ ಮಂಡಳಿಗಳು.

ಈ ಪೋರ್ಟಲ್‌ಗಳಿಗೆ APIs ಇಲ್ಲ. ಅಥವಾ ಅವರಿಗೆ ಅದಾಖಲೀಕರಿಸಿದ, ದರ-ಸೀಮಿತ, ಅಥವಾ ಪ್ರವೇಶಕ್ಕೆ ಪಾವತಿಸುವ "ಆದ್ಯ ಪಾಲುದಾರರಿಗೆ" ನಿರ್ಬಂಧಿತ APIs ಇವೆ. ಡೇಟಾ HTML ನಲ್ಲಿ ರೆಂಡರ್ ಮಾಡಿದ ಲಾಗಿನ್ ಪುಟದ ಹಿಂದೆ ಇರುತ್ತದೆ, ಮತ್ತು ಅದನ್ನು ಹೊರತೆಗೆಯಲು ಏಕೈಕ ಮಾರ್ಗ ಲಾಗ್ ಇನ್ ಮಾಡಿ UI ನ್ಯಾವಿಗೇಟ್ ಮಾಡುವುದು.

ಸಾಂಪ್ರದಾಯಿಕ ಸ್ವಯಂಚಾಲನ ಬ್ರೌಸರ್ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ಬಳಸುತ್ತದೆ. Selenium, Playwright, ಅಥವಾ Puppeteer ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು ಲಾಗ್ ಇನ್ ಮಾಡಿ, ಸರಿಯಾದ ಪುಟಕ್ಕೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ, CSS ಸೆಲೆಕ್ಟರ್ ಅಥವಾ XPath ಮೂಲಕ ಅಂಶಗಳನ್ನು ಕಂಡುಹಿಡಿದು, ಡೇಟಾ ಹೊರತೆಗೆದು, ಮತ್ತು ಲಾಗ್ ಔಟ್ ಮಾಡುತ್ತವೆ. ಈ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು ತಮ್ಮ ಕೆಲಸ ಮಾಡುವವರೆಗೆ ಕೆಲಸ ಮಾಡುತ್ತವೆ. ಪೋರ್ಟಲ್ ಮರು-ವಿನ್ಯಾಸ CSS ಕ್ಲಾಸ್ ಹೆಸರುಗಳನ್ನು ಬದಲಾಯಿಸುತ್ತದೆ. ಲಾಗಿನ್ ಹರಿವಿಗೆ ಹೊಸ CAPTCHA ಸೇರಿಸಲಾಗುತ್ತದೆ. ನ್ಯಾವಿಗೇಶನ್ ಮೆನು ಸೈಡ್‌ಬಾರ್‌ನಿಂದ ಹ್ಯಾಂಬರ್ಗರ್ ಮೆನುಗೆ ಸ್ಥಳಾಂತರಗೊಳ್ಳುತ್ತದೆ. ಸ್ಕ್ರಿಪ್ಟ್ ಮೌನವಾಗಿ ಮುರಿಯುತ್ತದೆ.

ರಾಜ್ಯ ವೈದ್ಯಕೀಯ ಮಂಡಳಿಗಳು ವಿಶೇಷವಾಗಿ ಕಠಿಣ ಉದಾಹರಣೆ. ಐವತ್ತು ಇದ್ದು, ಪ್ರತಿಯೊಂದೂ ಬೇರೆ ವೆಬ್‌ಸೈಟ್, ಬೇರೆ ಲೇಔಟ್, ಬೇರೆ ದೃಢೀಕರಣ ವಿಧಾನಗಳು, ಮತ್ತು ಬೇರೆ ಡೇಟಾ ಸ್ವರೂಪಗಳೊಂದಿಗೆ. ಅವು ಯಾವ ಸೂಚನೆ ಇಲ್ಲದೆ ತಮ್ಮದೇ ವೇಳಾಪಟ್ಟಿಯಲ್ಲಿ ಮರು-ವಿನ್ಯಾಸ ಮಾಡುತ್ತವೆ.

## Triggerfish ಇದನ್ನು ಹೇಗೆ ಪರಿಹರಿಸುತ್ತದೆ

Triggerfish ನ ಬ್ರೌಸರ್ ಸ್ವಯಂಚಾಲನ LLM-ಆಧಾರಿತ ದೃಶ್ಯ ನ್ಯಾವಿಗೇಶನ್‌ನೊಂದಿಗೆ CDP-ನಿಯಂತ್ರಿತ Chromium ಸಂಯೋಜಿಸುತ್ತದೆ. ಏಜೆಂಟ್ ಪುಟವನ್ನು ರೆಂಡರ್ ಮಾಡಿದ ಪಿಕ್ಸೆಲ್‌ಗಳು ಮತ್ತು ಪ್ರವೇಶಿಸುವ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್‌ಗಳಾಗಿ ನೋಡುತ್ತದೆ, DOM ಮರವಾಗಿ ಅಲ್ಲ. ಅದು ಅಂಶಗಳನ್ನು ಅವು ಹೇಗೆ ಕಾಣಿಸುತ್ತವೆ ಮತ್ತು ಏನು ಮಾಡುತ್ತವೆ ಎಂಬ ಆಧಾರದ ಮೇಲೆ ಗುರುತಿಸುತ್ತದೆ, CSS ಕ್ಲಾಸ್ ಹೆಸರಿನ ಮೇಲೆ ಅಲ್ಲ. ಪೋರ್ಟಲ್ ಮರು-ವಿನ್ಯಾಸ ಮಾಡಿದಾಗ, ಏಜೆಂಟ್ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ ಏಕೆಂದರೆ ಲಾಗಿನ್ ಫಾರ್ಮ್‌ಗಳು ಇನ್ನೂ ಲಾಗಿನ್ ಫಾರ್ಮ್‌ಗಳ ಹಾಗೆ ಕಾಣುತ್ತವೆ.

### ಸೆಲೆಕ್ಟರ್ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳ ಬದಲು ದೃಶ್ಯ ನ್ಯಾವಿಗೇಶನ್

ಬ್ರೌಸರ್ ಸ್ವಯಂಚಾಲನ ಟೂಲ್‌ಗಳು ಏಳು ಕಾರ್ಯಾಚರಣೆಗಳ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ: navigate, snapshot, click, type, select, scroll, ಮತ್ತು wait. ಏಜೆಂಟ್ URL ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡುತ್ತದೆ, ರೆಂಡರ್ ಮಾಡಿದ ಪುಟದ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ, ಅದು ನೋಡುವುದರ ಬಗ್ಗೆ ತರ್ಕ ಮಾಡುತ್ತದೆ, ಮತ್ತು ಯಾವ ಕ್ರಿಯೆ ತೆಗೆದುಕೊಳ್ಳಬೇಕೆಂದು ನಿರ್ಧರಿಸುತ್ತದೆ. ಪುಟ ಸಂದರ್ಭದಲ್ಲಿ ಅನಿಯಂತ್ರಿತ JavaScript ಚಲಿಸುವ `evaluate` ಟೂಲ್ ಇಲ್ಲ. ಏಜೆಂಟ್ ಪುಟದೊಂದಿಗೆ ಮಾನವ ಮಾಡುವ ರೀತಿಯಲ್ಲಿ, UI ಮೂಲಕ ಸಂವಾದ ಮಾಡುತ್ತದೆ, ಮತ್ತು ದುರ್ಭಾವಪೂರಿತ ಪುಟದಿಂದ ಶೋಷಿಸಬಹುದಾದ ಕೋಡ್ ಕಾರ್ಯಗತಗೊಳಿಸಲಾಗುವುದಿಲ್ಲ.

### ಹಂಚಿಕೆ ಡೊಮೇನ್ ಸುರಕ್ಷತೆ

ಬ್ರೌಸರ್ ನ್ಯಾವಿಗೇಶನ್ ವೆಬ್ ಫೆಚ್ ಕಾರ್ಯಾಚರಣೆಗಳಂತೆಯೇ ಅದೇ ಡೊಮೇನ್ ಸುರಕ್ಷತಾ ಸಂರಚನೆ ಹಂಚಿಕೊಳ್ಳುತ್ತದೆ. `triggerfish.yaml` ನಲ್ಲಿ ಒಂದೇ ಸಂರಚನಾ ಬ್ಲಾಕ್ SSRF ಡೆನಿಲಿಸ್ಟ್‌ಗಳು, ಡೊಮೇನ್ ಅಲೋಲಿಸ್ಟ್‌ಗಳು, ಡೊಮೇನ್ ಡೆನಿಲಿಸ್ಟ್‌ಗಳು, ಮತ್ತು ಡೊಮೇನ್-ವರ್ಗೀಕರಣ ಮ್ಯಾಪಿಂಗ್‌ಗಳನ್ನು ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ. ಏಜೆಂಟ್ CONFIDENTIAL ನಲ್ಲಿ ವರ್ಗೀಕರಿಸಿದ ವೆಂಡರ್ ಪೋರ್ಟಲ್‌ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿದಾಗ, ಅಧಿವೇಶನ taint ಸ್ವಯಂಚಾಲಿತವಾಗಿ CONFIDENTIAL ಗೆ ಹೆಚ್ಚಾಗುತ್ತದೆ.

SSRF ಡೆನಿಲಿಸ್ಟ್ ಹಾರ್ಡ್‌ಕೋಡ್ ಮಾಡಿದ ಮತ್ತು ಅತಿಕ್ರಮಿಸಲಾಗದ. ಖಾಸಗಿ IP ಶ್ರೇಣಿಗಳು, ಲಿಂಕ್-ಲೋಕಲ್ ವಿಳಾಸಗಳು, ಮತ್ತು ಕ್ಲೌಡ್ ಮೆಟಾಡೇಟಾ ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳು ಯಾವಾಗಲೂ ತಡೆಯಲ್ಪಡುತ್ತವೆ. DNS ಪರಿಹಾರ ವಿನಂತಿ ಮೊದಲು ಪರಿಶೀಲಿಸಲ್ಪಡುತ್ತದೆ, DNS ರಿಬೈಂಡಿಂಗ್ ದಾಳಿಗಳನ್ನು ತಡೆಯುತ್ತದೆ.

### ಬ್ರೌಸರ್ ಪ್ರೊಫೈಲ್ ವಾಟರ್‌ಮಾರ್ಕಿಂಗ್

ಪ್ರತಿ ಏಜೆಂಟ್ ತನ್ನದೇ ಬ್ರೌಸರ್ ಪ್ರೊಫೈಲ್ ನಿರ್ವಹಿಸುತ್ತದೆ, ಕಾಲಾನಂತರದಲ್ಲಿ ಪೋರ್ಟಲ್‌ಗಳೊಂದಿಗೆ ಸಂವಾದ ಮಾಡುತ್ತ ಕುಕೀಗಳು, ಲೋಕಲ್ ಸ್ಟೋರೇಜ್, ಮತ್ತು ಅಧಿವೇಶನ ಡೇಟಾ ಸಂಗ್ರಹಿಸುತ್ತದೆ. ಪ್ರೊಫೈಲ್ ವರ್ಗೀಕರಣ ವಾಟರ್‌ಮಾರ್ಕ್ ಒಯ್ಯುತ್ತದೆ. ಏಜೆಂಟ್ CONFIDENTIAL ವೆಂಡರ್ ಪೋರ್ಟಲ್‌ಗೆ ಲಾಗ್ ಇನ್ ಮಾಡಲು ಪ್ರೊಫೈಲ್ ಬಳಸಿದ್ದರೆ, ಪ್ರೊಫೈಲ್ CONFIDENTIAL ನಲ್ಲಿ ವಾಟರ್‌ಮಾರ್ಕ್ ಮಾಡಲ್ಪಡುತ್ತದೆ. PUBLIC ವರ್ಗೀಕರಣದಲ್ಲಿ ನಡೆಯುವ ನಂತರದ ಅಧಿವೇಶನ ಆ ಪ್ರೊಫೈಲ್ ಬಳಸಲಾಗುವುದಿಲ್ಲ.

### ರುಜುವಾತು ನಿರ್ವಹಣೆ

ಪೋರ್ಟಲ್ ರುಜುವಾತುಗಳನ್ನು OS keychain (ವೈಯಕ್ತಿಕ ಹಂತ) ಅಥವಾ ಎಂಟರ್‌ಪ್ರೈಸ್ vault (ಎಂಟರ್‌ಪ್ರೈಸ್ ಹಂತ) ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ, ಸಂರಚನಾ ಫೈಲ್‌ಗಳು ಅಥವಾ ಪರಿಸರ ಅಸ್ಥಿರಗಳಲ್ಲಿ ಎಂದಿಗೂ ಅಲ್ಲ. SECRET_ACCESS hook ಪ್ರತಿ ರುಜುವಾತು ಮರುಪಡೆಯುವಿಕೆ ದಾಖಲಿಸುತ್ತದೆ. ರುಜುವಾತುಗಳನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸುವ ಸಮಯದಲ್ಲಿ ಒದಗಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಟೈಪಿಂಗ್ ಇಂಟರ್‌ಫೇಸ್ ಮೂಲಕ ಬ್ರೌಸರ್ ಅಧಿವೇಶನಗಳಿಗೆ ಚುಚ್ಚಲಾಗುತ್ತದೆ.

### ಸಾಮಾನ್ಯ ಪೋರ್ಟಲ್ ಬದಲಾವಣೆಗಳ ವಿರುದ್ಧ ಸ್ಥಿತಿಸ್ಥಾಪಕತ್ವ

ಸಾಮಾನ್ಯ ಪೋರ್ಟಲ್ ಬದಲಾವಣೆಗಳು ಸಂಭವಿಸಿದಾಗ ಏನಾಗುತ್ತದೆ:

**ಲಾಗಿನ್ ಪುಟ ಮರು-ವಿನ್ಯಾಸ.** ಏಜೆಂಟ್ ಹೊಸ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ, ನವೀಕರಿಸಿದ ಲೇಔಟ್ ಗುರುತಿಸುತ್ತದೆ, ಮತ್ತು ದೃಶ್ಯ ಸಂದರ್ಭದಿಂದ ಫಾರ್ಮ್ ಕ್ಷೇತ್ರಗಳನ್ನು ಕಂಡುಹಿಡಿಯುತ್ತದೆ. ಪೋರ್ಟಲ್ ಸಂಪೂರ್ಣ ಬೇರೆ ದೃಢೀಕರಣ ವಿಧಾನಕ್ಕೆ (SAML, OAuth, ಹಾರ್ಡ್‌ವೇರ್ ಟೋಕನ್) ಬದಲಾಯಿಸದ ಹೊರತು, ಯಾವ ಸಂರಚನಾ ಬದಲಾವಣೆ ಇಲ್ಲದೆ ಲಾಗಿನ್ ಕಾರ್ಯ ನಿರ್ವಹಿಸುತ್ತದೆ.

**ನ್ಯಾವಿಗೇಶನ್ ಪುನರ್‌ರಚನೆ.** ಏಜೆಂಟ್ ಲಾಗಿನ್ ನಂತರ ಪುಟ ಓದುತ್ತದೆ ಮತ್ತು URL ಮಾದರಿಗಳ ಬದಲು ಲಿಂಕ್ ಪಠ್ಯ, ಮೆನು ಲೇಬಲ್‌ಗಳು, ಮತ್ತು ಪುಟ ಶೀರ್ಷಿಕೆಗಳ ಆಧಾರದ ಮೇಲೆ ಗುರಿ ವಿಭಾಗಕ್ಕೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡುತ್ತದೆ.

**ಹೊಸ ಕುಕಿ ಸಮ್ಮತಿ ಬ್ಯಾನರ್.** ಏಜೆಂಟ್ ಬ್ಯಾನರ್ ನೋಡುತ್ತದೆ, ಸ್ವೀಕಾರ/ವಜಾ ಬಟನ್ ಗುರುತಿಸುತ್ತದೆ, ಅದರ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡುತ್ತದೆ, ಮತ್ತು ಮೂಲ ಕಾರ್ಯದೊಂದಿಗೆ ಮುಂದುವರಿಯುತ್ತದೆ.

**CAPTCHA ಸೇರಿಸಲಾಗಿದೆ.** ಇಲ್ಲಿ ವಿಧಾನ ಪ್ರಾಮಾಣಿಕ ಮಿತಿಗಳನ್ನು ಹೊಂದಿದೆ. ಸರಳ ಚಿತ್ರ CAPTCHAs LLM ನ ದೃಷ್ಟಿ ಸಾಮರ್ಥ್ಯಗಳ ಆಧಾರದ ಮೇಲೆ ಪರಿಹರಿಸಬಹುದು, ಆದರೆ reCAPTCHA v3 ಮತ್ತು ಇದೇ ರೀತಿಯ ವ್ಯವಹಾರ ವಿಶ್ಲೇಷಣಾ ವ್ಯವಸ್ಥೆಗಳು ಸ್ವಯಂಚಾಲಿತ ಬ್ರೌಸರ್‌ಗಳನ್ನು ತಡೆಯಬಹುದು. ವರ್ಕ್‌ಫ್ಲೋ ಮೌನವಾಗಿ ವಿಫಲವಾಗುವ ಬದಲು ಇವುಗಳನ್ನು ಮಾನವ ಹಸ್ತಕ್ಷೇಪ ಸರದಿಗೆ ರೂಟ್ ಮಾಡುತ್ತದೆ.

**MFA ಪ್ರಾಂಪ್ಟ್‌ಗಳು.** ಪೋರ್ಟಲ್ ಮೊದಲು ಅಗತ್ಯ ಇರದ MFA ಅಗತ್ಯ ಮಾಡಿದರೆ, ಏಜೆಂಟ್ ಅನಿರೀಕ್ಷಿತ ಪುಟ ಪತ್ತೆ ಮಾಡುತ್ತದೆ, ಅಧಿಸೂಚನಾ ವ್ಯವಸ್ಥೆ ಮೂಲಕ ಪರಿಸ್ಥಿತಿ ವರದಿ ಮಾಡುತ್ತದೆ, ಮತ್ತು ಮಾನವ MFA ಹಂತ ಪೂರ್ಣ ಮಾಡುವವರೆಗೆ ವರ್ಕ್‌ಫ್ಲೋ ನಿಲ್ಲಿಸುತ್ತದೆ.

### ಹಲವಾರು ಪೋರ್ಟಲ್‌ಗಳಲ್ಲಿ ಬ್ಯಾಚ್ ಸಂಸ್ಕರಣೆ

ವರ್ಕ್‌ಫ್ಲೋ ಎಂಜಿನ್‌ನ `for` ಲೂಪ್ ಬೆಂಬಲ ಒಂದೇ ವರ್ಕ್‌ಫ್ಲೋ ಹಲವಾರು ಪೋರ್ಟಲ್ ಗುರಿಗಳ ಮೇಲೆ iterate ಮಾಡಬಲ್ಲದು ಎಂದು ಅರ್ಥ. ರುಜುವಾತು ಪರಿಶೀಲನಾ ಸೇವೆ ಒಂದೇ ಬ್ಯಾಚ್ ರನ್‌ನಲ್ಲಿ ಐವತ್ತೂ ರಾಜ್ಯ ವೈದ್ಯಕೀಯ ಮಂಡಳಿಗಳಲ್ಲಿ ಪರವಾನಗಿ ಸ್ಥಿತಿ ಪರಿಶೀಲಿಸುವ ವರ್ಕ್‌ಫ್ಲೋ ವ್ಯಾಖ್ಯಾನಿಸಬಲ್ಲದು.

## ಅಭ್ಯಾಸದಲ್ಲಿ ಇದು ಹೇಗೆ ಕಾಣುತ್ತದೆ

ರುಜುವಾತು ಸಂಸ್ಥೆ ಪ್ರದಾತ ನೋಂದಣಿ ಪ್ರಕ್ರಿಯೆಯ ಭಾಗವಾಗಿ ರಾಜ್ಯ ವೈದ್ಯಕೀಯ ಮಂಡಳಿಗಳಲ್ಲಿ ಆರೋಗ್ಯ ರಕ್ಷಣೆ ಪ್ರದಾತ ಪರವಾನಗಿಗಳನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ. ಸಾಂಪ್ರದಾಯಿಕವಾಗಿ, ಪರಿಶೀಲನಾ ಸಹಾಯಕರು ಪ್ರತಿ ಮಂಡಳಿ ವೆಬ್‌ಸೈಟ್‌ಗೆ ಹಸ್ತಚಾಲಿತವಾಗಿ ಲಾಗ್ ಇನ್ ಮಾಡಿ, ಪ್ರದಾತ ಹುಡುಕಿ, ಪರವಾನಗಿ ಸ್ಥಿತಿ ಸ್ಕ್ರೀನ್‌ಶಾಟ್ ತೆಗೆದುಕೊಂಡು, ರುಜುವಾತು ವ್ಯವಸ್ಥೆಗೆ ಡೇಟಾ ನಮೂದಿಸುತ್ತಾರೆ. ಪ್ರತಿ ಪರಿಶೀಲನೆ ಐದರಿಂದ ಹದಿನೈದು ನಿಮಿಷ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ.

Triggerfish ನೊಂದಿಗೆ, ವರ್ಕ್‌ಫ್ಲೋ ಸಂಪೂರ್ಣ ಪರಿಶೀಲನಾ ಚಕ್ರ ನಿರ್ವಹಿಸುತ್ತದೆ. ರಾಜ್ಯ ಮಂಡಳಿ ಪೋರ್ಟಲ್ ಮರು-ವಿನ್ಯಾಸ ಮಾಡಿದಾಗ, ಏಜೆಂಟ್ ಮುಂದಿನ ಪರಿಶೀಲನಾ ಪ್ರಯತ್ನದಲ್ಲಿ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ. CAPTCHA ಸ್ವಯಂಚಾಲಿತ ಪ್ರವೇಶ ತಡೆದಾಗ, ವರ್ಕ್‌ಫ್ಲೋ ಆ ರಾಜ್ಯವನ್ನು ಹಸ್ತಚಾಲಿತ ಪರಿಶೀಲನೆಗಾಗಿ ಫ್ಲ್ಯಾಗ್ ಮಾಡುತ್ತದೆ ಮತ್ತು ಉಳಿದ ಬ್ಯಾಚ್ ಸಂಸ್ಕರಣೆ ಮುಂದುವರಿಸುತ್ತದೆ.
