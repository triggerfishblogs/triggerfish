# सुरक्षा-प्रथम डिज़ाइन

Triggerfish एक एकल आधार पर बनाया गया है: **LLM के पास शून्य अधिकार है**। यह
कार्यों का अनुरोध करता है; policy परत निर्णय लेती है। प्रत्येक सुरक्षा निर्णय
निश्चयात्मक कोड द्वारा लिया जाता है जिसे AI बायपास, ओवरराइड, या प्रभावित नहीं
कर सकता।

## सुरक्षा LLM के नीचे क्यों होनी चाहिए

बड़े भाषा मॉडल prompt-inject किए जा सकते हैं। एक सावधानी से तैयार किया गया
इनपुट LLM को उसके निर्देशों की अनदेखी करने और ऐसे कार्य करने के लिए प्रेरित
कर सकता है जो उसे नहीं करने चाहिए। यदि आपका सुरक्षा मॉडल LLM पर नियमों का
पालन करने पर निर्भर करता है, तो एक सफल injection प्रत्येक सुरक्षा उपाय को
बायपास कर सकता है।

Triggerfish सभी सुरक्षा प्रवर्तन को एक कोड परत पर ले जाकर इसे हल करता है जो
LLM के **नीचे** बैठती है।

<img src="/diagrams/enforcement-layers.svg" alt="प्रवर्तन परतें: LLM के पास शून्य अधिकार, policy परत सभी निर्णय निश्चयात्मक रूप से लेती है" style="max-width: 100%;" />

::: warning सुरक्षा LLM परत के पास policy प्रवर्तन परत को ओवरराइड, छोड़ने, या
प्रभावित करने का कोई तंत्र नहीं है। कोई "LLM आउटपुट से बायपास कमांड पार्स
करें" तर्क नहीं है। अलगाव आर्किटेक्चरल है, व्यवहारिक नहीं। :::

## मूल अपरिवर्तनीय

> **समान इनपुट हमेशा समान सुरक्षा निर्णय उत्पन्न करता है। कोई यादृच्छिकता नहीं,
> कोई LLM कॉल नहीं, कोई विवेकाधिकार नहीं।**

## गहन रक्षा

Triggerfish रक्षा की तेरह परतें लागू करता है:

1. **चैनल प्रमाणीकरण** -- session स्थापना पर कोड-सत्यापित पहचान
2. **अनुमति-जागरूक डेटा एक्सेस** -- स्रोत सिस्टम अनुमतियाँ
3. **Session taint ट्रैकिंग** -- स्वचालित, अनिवार्य, केवल-वृद्धि
4. **डेटा lineage** -- प्रत्येक डेटा तत्व के लिए पूर्ण उत्पत्ति श्रृंखला
5. **Policy प्रवर्तन hooks** -- निश्चयात्मक, बायपास-रहित, लॉग किए गए
6. **MCP Gateway** -- प्रति-tool अनुमतियों के साथ सुरक्षित बाहरी tool एक्सेस
7. **Plugin sandbox** -- Deno + WASM दोहरा अलगाव
8. **Secrets अलगाव** -- OS keychain या vault
9. **Filesystem tool sandbox** -- पथ jail, पथ वर्गीकरण, taint-स्कोप्ड I/O
10. **Agent पहचान** -- क्रिप्टोग्राफ़िक delegation chains
11. **ऑडिट लॉगिंग** -- सभी निर्णय रिकॉर्ड, कोई अपवाद नहीं
12. **SSRF रोकथाम** -- IP denylist + DNS resolution जाँच
13. **Memory वर्गीकरण गेटिंग** -- writes session taint पर मजबूर, reads `canFlowTo` द्वारा फ़िल्टर
